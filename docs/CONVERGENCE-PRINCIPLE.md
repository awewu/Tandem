# 收敛原则 · Convergence Principle

> **「AI 给员工 3+1 选项, 不替员工决策」**
>
> 版本: v1.0
> 最后更新: 2026-05
> 性质: 牛马搭子核心方法论文档. 与 MANIFESTO 第二条、第三条直接对应.

---

## 摘要 (TL;DR)

收敛原则是牛马搭子区别于通用 AI 工具 (ChatGPT / AutoGen / Devin) 的产品哲学核心.

它由三个相互锚定的设计组成:

```
🎯 目标驱动收敛 (Target-Driven Convergence)
       ↓
🅰🅱🅲🅳 3+1 决策框架 (3+1 Decision Framework)
       ↓
⏱️ 议事室 5 步骨架 + 17 分钟硬上限 (Convergence Room)
```

**核心反对**:
- 多 Agent 互相辩论扩散
- AI 一键自动决策替代员工
- 无时间约束的"再讨论 1 轮"循环

**核心主张**:
- AI 输出始终锚定 KR/TTI 收敛
- 员工是决策主体, AI 是结构化辅助
- 时间是产品物理常数, 不可妥协

---

## 第一章: 为什么需要收敛原则

### 1.1 通用 AI 工具的"扩散陷阱"

主流 AI Agent 工具普遍陷入"扩散陷阱":

```
ChatGPT 派 (单 Agent 自由发散)
   ↓
   "让我想想..." → 给出 5 段分析 → 员工再问 → 再给 5 段
   ↓
   结果: 信息很多, 决策没有

AutoGen / CrewAI 派 (多 Agent 互相辩论)
   ↓
   Agent A: "我觉得方案 X..."
   Agent B: "但是 Y 风险..."
   Agent A: "Y 风险可以通过 Z 缓解..."
   Agent B: "Z 又会引入..."
   ↓
   结果: 辩论 30 轮, 员工更迷茫

Devin / 通用执行 Agent 派 (AI 直接代干)
   ↓
   "我已经为你做了 X, 提交了 PR"
   ↓
   结果: 员工不学习, 沦为橡皮图章
```

这些工具的根本问题是: **没有目标锚点, 没有时间约束, 没有明确的决策主体**.

### 1.2 牛马搭子的三重锚定

牛马搭子用三道铁律强制收敛:

| 锚点 | 作用 | 防止 |
|---|---|---|
| KR/TTI 锚定 | AI 输出必须关联具体目标 | 自由发散 |
| 3+1 选项 | AI 输出必须收敛到 4 个选项 | 多 Agent 辩论扩散 |
| 17 分钟上限 | 单议题强制收敛或升级 | 无限循环 |

> **「AI 不是讨论伙伴, AI 是收敛工具.」**

## 第二章: 3+1 决策框架

### 2.1 4 个选项的定义

任何 AI 决策辅助场景, 必须呈现以下 4 个选项 (3 个 AI 派别 + 1 个员工自创):

```
┌──────────────────────────────────────────────────┐
│             3+1 Decision Framework               │
├──────────────────────────────────────────────────┤
│                                                  │
│  🅰 SOP 方案                                     │
│     来源: Memory 层 SOP 库                       │
│     标签: 守正 / 制度派                          │
│     性格: "公司一直这么做"                       │
│                                                  │
│  🅱 推演方案                                     │
│     来源: 公司 Baseline 模型实时推演              │
│     标签: 出奇 / 理性派                          │
│     性格: "数据显示这样最优"                     │
│                                                  │
│  🅲 经验方案                                     │
│     来源: 案例库 (历史 Decision Cards)            │
│     标签: 借势 / 历史派                          │
│     性格: "类似情况上次这样做"                   │
│                                                  │
│  🅳 自创方案 (员工)                              │
│     来源: 员工本人, 强制填"我多看到了什么"        │
│     标签: 临场 / 直觉派                          │
│     性格: "这次有些不同, 因为..."                │
└──────────────────────────────────────────────────┘
```

### 2.2 每个选项的强制元数据

每个选项必须展示:

| 元数据 | 含义 | 示例 |
|---|---|---|
| 置信度 | AI 对该方案的信心 (0-1 或 高/中/低) | 0.78 / 高 |
| 风险评级 | 失败的潜在代价 | 低 / 中 / 高 |
| 适用边界 | 什么情况下不适用 | "客户规模 > 10 万时不适用" |
| 关联 KR/TTI | 影响哪些目标 | KR-001 (+12pp 预期) |
| 时间窗 | 实施所需时长 | 2 周 / 4 天 |
| 资源需求 | 人力 / 预算 | 2 人 × 1 周 |

### 2.3 🅳 自创方案的设计意图

🅳 选项是**反 AI 替代主义**的核心防御.

它的设计要求:
- **强制填写**: 员工必须填"我多看到了什么", 不能跳过
- **不评分**: AI 不打分员工方案 (避免心理打压)
- **可融合**: 员工可标注 "🅳 = 🅰 + 临场调整"
- **自创采纳激励**: 🅳 被采纳后会进入员工拿捏老板成长档案

> **🅳 不是为了让员工"自己想", 而是为了**保留员工的临场判断权**.**

### 2.4 例外场景: 常规决策可省略 🅱🅲🅳

不是所有决策都需要 3+1. **常规已知场景** (SOP 已完整覆盖) 可只给 🅰 + 一键确认:

| 场景 | 决策类型 | AI 输出 |
|---|---|---|
| 报销 ¥200 餐费 | 常规 | 仅 🅰 (按 SOP), 一键确认 |
| 预约 1:1 会议 | 常规 | 仅 🅰 (按日历), 一键确认 |
| 调整 KR 进度 ±5% | 常规 | 仅 🅰 (按周报), 一键确认 |
| 新客户合同条款 | 复杂 | 🅰🅱🅲🅳 全 4 选项 |
| 重大产品方向 | 战略 | 🅰🅱🅲🅳 + 议事室 |
| 跨部门资源调配 | 复杂 | 🅰🅱🅲🅳 + 议事室 |

**判断规则**:
- AI 自动识别决策类型 (常规 / 复杂 / 战略)
- 员工可手动升级为 3+1
- 战略级决策**强制**进入议事室

## 第三章: 议事室 5 步骨架

### 3.1 完整流程

议事室 (Convergence Room) 是基于 3+1 框架的**结构化会议形态**.

```
┌─────────────────────────────────────────────────────┐
│       议事室 5 步骨架 · 单议题 17 分钟硬上限          │
├─────────────────────────────────────────────────────┤
│                                                     │
│  1. ALIGN     校准    2 min                         │
│     • 锚定 KR/TTI                                   │
│     • 同步关键信息 (上一次决议状态)                  │
│     • 明确参会人角色                                │
│     ↓                                               │
│                                                     │
│  2. FRAME     界定    3 min                         │
│     • 一句话陈述决策问题                            │
│     • AI 识别决策类型 (常规/复杂/战略)              │
│     • 锁定决策原则 (谁拍板, 多数 vs 一致)           │
│     ↓                                               │
│                                                     │
│  3. DIVERGE   发散    5 min                         │
│     • AI 生成 🅰🅱🅲 三方案                          │
│     • 每位参会人填写 🅳 (自己的视角)                 │
│     • 5 分钟硬阀, 不可扩展                          │
│     ↓                                               │
│                                                     │
│  4. CONVERGE  收敛    4 min                         │
│     • 投票表决 (匿名或实名)                         │
│     • 分歧检测 → AI 暴露根源                        │
│     • 仍分歧 → 升级 (见 3.3)                        │
│     ↓                                               │
│                                                     │
│  5. COMMIT    落地    3 min                         │
│     • Decision Card 自动生成                        │
│     • 行动项 + Owner + Due Date                     │
│     • 后果回溯指针 (review_at 设定)                 │
│                                                     │
│  ─────────────────────────────────────────────     │
│   合计 17 分钟. 超时 → 议题外溢 → 单独排会议        │
└─────────────────────────────────────────────────────┘
```

### 3.2 17 分钟硬上限的设计逻辑

为什么是 17 分钟? 不是 15、不是 20?

**17 分钟 = 5 步最优配比**:
- ALIGN 2 min: 同步信息已被预读 (AI 提供 brief), 现场只需确认
- FRAME 3 min: 一句话陈述足够; 决策类型识别 AI 协助
- DIVERGE 5 min: 经验数据显示, 5 分钟是高质量发散的甜区 (短了仓促, 长了开始重复)
- CONVERGE 4 min: 投票 1 min + 分歧讨论 3 min
- COMMIT 3 min: AI 自动生成 Decision Card, 人类只需 review

**心理学依据**:
- 人类专注的"小段"约 12-20 分钟 (Pomodoro 25 是上限)
- 17 分钟略短于 Pomodoro, 留出场前/场后切换 buffer
- 一天可开 4-6 个议事室不疲劳

**反例**:
- 30 分钟会议: 前 17 分钟讨论, 后 13 分钟扯淡 (帕金森定律)
- 60 分钟会议: 实际有效讨论 ≤ 20 分钟, 其余是冗余

> **会议时长扩展不带来更好决策, 只带来更多消耗.**

### 3.3 收敛失败的升级机制

议事室的灵魂在于"分歧无法收敛时不强求, 直接升级".

```
CONVERGE 步骤失败的处理逻辑:

第一轮投票 → 分歧 > 阈值
  ↓
AI 暴露分歧根源 (例: "🅰 vs 🅱 的核心分歧是: A 认为 X, B 认为 Y")
  ↓
再讨论 1 轮 (1 min, 仅暴露分歧, 不重新发散)
  ↓
第二轮投票
  ↓
仍分歧 > 阈值
  ↓
🚨 自动升级
  ↓
通知"决策人" (议事室创建时指定的上级 / 委员会)
  ↓
决策人收到结构化分歧摘要 (5 分钟可读完)
  • 议题 1 句话
  • 🅰🅱🅲🅳 4 方案概要
  • 每方支持人 + 关键论点
  • AI 推荐 (可选, 标注信心)
  ↓
决策人异步拍板 (≤ 24h)
  ↓
Decision Card 生成, 标注 "由 [决策人] 升级裁决"
```

**关键铁律**:

- 议事室**不调解分歧**, 只暴露分歧
- AI **不投票**, AI 只展示选项
- 升级**不是失败**, 是合理的责任传递

## 第四章: AI 在 5 步中的角色细则

### 4.1 AI 职责矩阵

| 步骤 | AI 做什么 | AI 不做什么 |
|---|---|---|
| ALIGN | 提供前情摘要 / 关联 KR 状态 | 不替主持人开场 |
| FRAME | 识别决策类型 / 提示原则 | 不替员工陈述问题 |
| DIVERGE | 生成 🅰🅱🅲 / 检索 SOP/案例 | 不评分 🅳 自创方案 |
| CONVERGE | 检测分歧 / 暴露根源 | 不投票 / 不站队 / 不调解 |
| COMMIT | 生成 Decision Card 草稿 | 不锁定行动项 (员工确认后才锁) |

### 4.2 AI 的"沉默时刻"

议事室设计了多个**强制 AI 沉默时刻**, 防止 AI 影响人类判断:

```
DIVERGE 步骤:
  • AI 提供 🅰🅱🅲 后立即闭麦
  • 员工填 🅳 时, AI 不再插话
  • 员工自创时, AI 不打分

CONVERGE 步骤:
  • 投票时, AI 不显示 "AI 推荐"
  • 等人类投完票, AI 才暴露分歧
  • AI 不发表"我倾向哪个"

COMMIT 步骤:
  • Decision Card 草稿生成后, 员工 review
  • 员工不点确认, AI 不锁定
```

**为什么需要沉默时刻**:
- AI 总在场会让员工产生"AI 比我聪明"的依赖心理
- 沉默时刻保护员工的独立判断权
- 也是「员工成长」承诺的具体落地

## 第五章: 与同类决策框架对比

### 5.1 对比表

| 框架 | 决策主体 | 选项数 | 时间约束 | 升级机制 |
|---|---|---|---|---|
| ChatGPT 单 Agent | 模糊 (员工自己拼) | 不限 | 无 | 无 |
| AutoGen 多 Agent | AI 群体 | 不限 (扩散) | 无 | 无 |
| Devin 执行 Agent | AI | 1 (直接执行) | 无 | 无 |
| Microsoft Copilot | 员工 | 1 (推荐) | 无 | 无 |
| RICE / ICE 框架 | 员工 | 不限 (打分) | 弱 | 无 |
| OKR 经典 | 员工 | 不限 | 季度 | 弱 |
| **牛马搭子 (Tandem)** | **员工 + 升级路径** | **3+1** | **17 min/24h** | **强 (议事室自带)** |

### 5.2 牛马搭子的差异化

牛马搭子在三个维度上独特:

1. **3+1 是收敛的, 不是发散的**
   - 4 个选项一次性给出, 不会"再来 5 个"
   - 来源不同 (SOP / 推演 / 案例 / 自创), 视角天然差异化
   - 员工选择即决策, 不需要再筛选

2. **17 分钟是物理常数, 不是建议**
   - 时间约束写在产品里, 不能配置
   - 超时强制升级, 不强制延长
   - 反帕金森定律的设计

3. **升级是设计内, 不是兜底**
   - 议事室创建时强制指定"决策人"
   - 升级路径自动触发, 不需人工召集
   - 决策人收到的是结构化摘要, 不是冗长聊天记录

## 第六章: 决策类型分级

### 6.1 三档决策

不同决策应用不同等级的 3+1 框架:

```
档位 A: 常规决策 (Routine)
  • SOP 完整覆盖
  • AI 仅给 🅰, 一键确认
  • 时长: < 1 分钟
  • 例: 报销 / 排假 / 例行审批

档位 B: 复杂决策 (Complex)
  • 多方案权衡
  • AI 给 🅰🅱🅲🅳 全 4 选项
  • 时长: 5-15 分钟 (个人) / 17 分钟 (议事室)
  • 例: 灰度策略 / 资源调配 / 客户方案

档位 C: 战略决策 (Strategic)
  • 影响 KR 大方向
  • 强制议事室 + 升级路径
  • 时长: 17 分钟 + 24h 升级窗口
  • 例: 产品方向 / 组织调整 / 重大投入
```

### 6.2 自动识别规则

AI 用以下信号自动识别决策档位:

| 信号 | 档位 |
|---|---|
| SOP 命中 + 历史数据丰富 | A 常规 |
| 影响 ≥ 1 个 KR | B 复杂 |
| 影响 ≥ 1 个 KPI 关键指标 | B 复杂 |
| 涉及预算 ≥ 阈值 | B 复杂 |
| 涉及人事调动 | C 战略 |
| 影响公司级 OKR 直系 | C 战略 |
| 涉及红线 (合规/价值观) | C 战略 |

员工可手动升级或降级, 但**降级到 A 档需治理官批准** (避免规避议事室).

## 第七章: Decision Card 产出规范

### 7.1 必备字段

每个议事室结束必须产出 1 个 Decision Card. 字段:

```yaml
id: DC-2026-Q2-7723
created_via: convergence_room   # 或 individual_3+1
title: "用户邀请奖励 v2 上线策略"

# 议事室元数据
room_id: CR-9981
room_duration_min: 16            # 实际时长 (≤ 17)
participants: [李四, 王五, 张三]
decision_maker: 王五              # 升级时填决策人

# 锚点
related_kr: [KR-005]
related_tti: [TTI-012]
decision_class: complex          # routine / complex / strategic

# 4 个选项
options:
  A_sop:        {...}
  B_reasoning:  {...}
  C_historical: {...}
  D_original:   {...}

# 收敛结果
selected: D
selected_by: 李四
vote_record:
  - {voter: 李四, choice: D}
  - {voter: 王五, choice: A}
  - {voter: 张三, choice: D}
divergence_score: 0.33           # 0-1, 越高越分歧
escalated: false

# 行动项
action_items:
  - {owner: 李四, task: "...", due: "2026-04-17"}

# 回溯
retrospective_pointer:
  review_at: 2026-05-01
  actual_outcome: null

# 水印
watermark:
  by_persona: false
  confirmed_by_owner: true
```

### 7.2 自动归档

Decision Card 生成后自动:

- 写入 Material 层 (会议纪要库)
- 关联到 KR/TTI 仪表盘
- 推送给 action_items 中的 Owner
- 设定 retrospective 提醒
- **不会**自动写入 Memory 层 (Memory 必须签批)

## 第八章: 实施指南

### 8.1 对员工

新员工 5 分钟培训即可上手:

```
1. 看到 AI 给的 4 个选项, 不要直接选 🅰
2. 先扫一眼 🅱🅲, 看看是否启发
3. 想想 🅳: 这次有什么不同?
4. 选一个, 加一句"为什么选这个" (强制)
5. 行动项确认, 完成

需要议事室时:
1. 点"开议事室", 选议题 + 决策人
2. 邀请相关人 (≤ 5 人最佳)
3. 进入 5 步骨架, 跟着流程走
4. 17 分钟到了, 该升级就升级, 别硬撑
```

### 8.2 对管理者

管理者最需要适应的是:

- ✅ 接受"AI 不替我决策, AI 给我 4 个选项让我选"
- ✅ 接受"分歧不一定要现场调解, 升级是合法路径"
- ✅ 接受"17 分钟不够就升级, 不要无限延会"
- ❌ 不要把议事室当传统会议开 (1 小时议程)
- ❌ 不要要求 AI "替我推荐一个"
- ❌ 不要把决策人指定为自己以外的人 (推卸责任)

### 8.3 对治理官

治理官每季度审视:

- 议事室平均时长是否在 17 分钟内? (超时率 > 20% 需诊断)
- 升级率是否正常? (5-15% 是健康区间)
- 🅳 自创方案采纳率? (> 30% 是健康)
- Decision Card 质量? (回溯 review 是否真的填了)

## 第九章: 反例与禁忌

### 9.1 禁止的产品模式

| 反例 | 为何禁止 |
|---|---|
| AI "再来 5 个方案" | 违反 3+1 收敛原则 |
| AI 在投票前显示"AI 推荐" | 影响人类独立判断 |
| 议事室时长可配置为 30 / 60 分钟 | 违反 17 分钟物理常数 |
| 多 Agent 互相辩论给员工看 | 违反目标驱动收敛 |
| AI 直接执行 (无员工确认) | 违反员工是决策主体 |
| 🅳 自创方案打分排序 | 心理打压, 违反尊重 |
| 升级摘要超过 1 屏 | 违反"决策人 5 分钟可读" |

### 9.2 文化反例

| 反例行为 | 应对 |
|---|---|
| 员工总是无脑选 🅰 (不动脑子) | 拿捏老板提示"已连续 N 次选 🅰, 请关注 🅳" |
| 经理压员工"按 AI 推荐" | 培训 + 治理官介入 |
| 议事室开成传统会议 | 强制 17 分钟到点关闭 |
| 决策人长期不接升级 | 自动 +1 级 escalate |

---

## 结语: 收敛是 AI 时代的纪律

通用 AI 工具的扩散是时代的早期幼稚.

牛马搭子相信: **AI 越强大, 越需要纪律**.

3+1 框架、17 分钟上限、议事室 5 步骨架, 这三个看似"限制" AI 能力的设计, 实际上**让 AI 真正可用**:

- 员工知道 AI 会给什么 → 信任产生
- 员工知道时间会到边界 → 心安产生
- 员工知道决策权在自己 → 成长产生

> **「让 AI 收敛, 让员工生长.」**
