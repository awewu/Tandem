# 分身代参会议设计 · Meeting Proxy

> **「员工有空时本人在场, 员工不在时分身代劳, 员工随时可一键收回授权.」**
>
> 版本: v1.0
> 最后更新: 2026-05
> 性质: 牛马搭子核心差异化功能设计文档

---

## 摘要 (TL;DR)

分身代参 (Meeting Proxy) 是牛马搭子最具传播力的 PLG 杀手特性.

**一句话定位**:
> 「飞书让你**永远在线**. 牛马搭子让你**该在的时候在, 不该在的时候有人替你**.」

**核心设计**:

- **不自建会议系统**: 寄生腾讯会议等基础设施
- **微信式选择**: 员工每次决定本人参 / 分身代参 / 拒绝
- **控制中枢**: 员工对会议纪要的事后确认 (而非 SKU gate)
- **强制水印**: 分身入会必显式标识, 反 AI 欺诈
- **红区禁用**: 客户/合规/招聘等敏感会议系统强制禁分身

## 第一章: 产品哲学

### 1.1 微信式选择的洞察

```
┌────────────────────────────────────────────────────────┐
│        微信的天才设计 vs 牛马搭子的对应升级             │
├────────────────────────────────────────────────────────┤
│                                                        │
│  微信               牛马搭子                            │
│  ──────────         ──────────                         │
│  📞 视频通话    →   👤 本人参会 (同步)                  │
│  📱 语音消息    →   🤖 分身代参 (异步代理)              │
│  💬 文字消息    →   📝 文字反馈 (Decision Card)         │
│                                                        │
│  共同哲学:                                             │
│  ★ 让用户根据"当下状态"选择最合适的参与形式            │
│  ★ 不预设"必须实时同步"才算认真                        │
│  ★ 让通信工具适应人, 不是人适应工具                    │
└────────────────────────────────────────────────────────┘
```

### 1.2 解决的真实痛点

| 痛点 | 现状 | 牛马搭子方案 |
|---|---|---|
| 会议过多, 心流被切碎 | 每天 5-8 个会, 无空隙工作 | 70% 可代参, 留出心流时段 |
| 8 小时外仍要响应会议 | "下班后再开个会"成习惯 | 分身代参, 员工真正下班 |
| 跨时区协作疲劳 | 凌晨 3 点参会 | 分身代参 + 24h 否决窗口 |
| 病假/产假/陪护无替代 | 工作丢失 / 同事代开 | 分身代参延续工作 |
| 述职准备耗时 | 加班一周写 PPT | AI 自动生成述职包 |

### 1.3 「四个满意」的体现

| 满意 | 分身代参的价值 |
|---|---|
| 客户满意 | 会议有人参加, 决议正常推进 |
| 员工满意 | 8 小时外不被打扰, 心流被保护 |
| 股东满意 | 团队效能提升, 会议成本下降 |
| 社会满意 | 推动职场反卷, 重新定义"在场" |

## 第二章: 架构原则

### 2.1 不自建会议系统

**铁律**: 牛马搭子永不自建视频会议基础设施.

理由:

- 腾讯会议每年研发投入数十亿, 无法竞争
- 我们的价值在**智能层**, 不在底座
- 自建会议 = 重复造轮子, 资源耗散

### 2.2 薄壳寄生

```
┌──────────────────────────────────────────────────────┐
│                牛马搭子 = 会议智能层                  │
├──────────────────────────────────────────────────────┤
│                                                      │
│   ⬇  调用                                            │
│  ┌────────────────────────────────────────────┐     │
│  │  腾讯会议 (基础设施)                        │     │
│  │  • 视频/音频底座                            │     │
│  │  • 录制 / 字幕                              │     │
│  │  • 屏幕共享 / 白板                          │     │
│  │  → 客户用腾讯会议企业版, 该付费付费         │     │
│  └────────────────────────────────────────────┘     │
│                                                      │
│   ⬆  注入智能                                        │
│  ┌────────────────────────────────────────────┐     │
│  │  牛马搭子智能层                             │     │
│  │  • 会议邀请决策 (本人 / 分身 / 拒绝)        │     │
│  │  • 分身代参 (语音 + 决策)                   │     │
│  │  • 实时议程导航 (议事室 5 步)               │     │
│  │  • 自动生成 Decision Card                   │     │
│  │  • 自动入 Material 层                       │     │
│  └────────────────────────────────────────────┘     │
└──────────────────────────────────────────────────────┘
```

### 2.3 V2/V3 多平台支持

V1: 仅腾讯会议
V2: + 飞书会议 + Zoom
V3: + Microsoft Teams + Google Meet

**协议薄壳化**, 对客户透明: 客户用什么会议, 牛马搭子接什么.

## 第三章: 用户体验流程 (5 步)

### 3.1 收到会议邀请的瞬间 (3 秒决策)

```
┌─────────────────────────────────────────────────┐
│ 📅 周一 14:00 - 14:30  (30 分钟)                │
│ 议题: Q2 用户增长策略评审                       │
│ 主持: 王经理                                    │
│ 参会: 你 + 5 人                                 │
│ 关联 KR: KR-001, KR-005                        │
│                                                 │
│ ─────────────────────────────────────────────  │
│ 你的搭子分析:                                   │
│ • 议题与你 Q2 KR 关联度: 高                     │
│ • 此时段你的日历: 空                            │
│ • 历史: 同类会议你曾分享 X 数据                 │
│                                                 │
│ ─────────────────────────────────────────────  │
│ 选择参会方式:                                   │
│                                                 │
│ ┌──────────────┐ ┌──────────────┐ ┌─────────┐ │
│ │ 👤 本人参加   │ │ 🤖 分身代参   │ │ ❌ 拒绝  │ │
│ │              │ │              │ │         │ │
│ │ (打开腾讯会议)│ │ (你下班/忙时) │ │ (说明)  │ │
│ └──────────────┘ └──────────────┘ └─────────┘ │
└─────────────────────────────────────────────────┘
```

设计要点:

- 3 秒决策: 像微信选语音/视频一样轻
- AI 推荐基于历史 + 日历 + KR 关联度
- 不强制选择, 默认是"本人参加"

### 3.2 分身代参授权卡 (会前)

每次代参不是无限授权, 是**"这一次会"的范围授权**:

```
┌─────────────────────────────────────────────────┐
│ 🤖 分身代参授权 - Q2 用户增长策略评审           │
│ ─────────────────────────────────────────────  │
│                                                 │
│ 你的分身将代你参加. 请确认授权范围:             │
│                                                 │
│ ☑ 可代为汇报 KR-001 进度                       │
│ ☑ 可代为分享上周决议 #DC-7723                   │
│ ☑ 可表态: 同意/反对 (基于你历史立场)            │
│ ☑ 可承诺动作: ≤ 1 工作日内完成                  │
│                                                 │
│ ☐ 不可代签 > ¥10,000 决议                       │
│ ☐ 不可代表态对人事评价                          │
│ ☐ 不可代承诺加班                                │
│                                                 │
│ 📌 关键议题分身将向你请示 (异步推送)             │
│ 📌 会后 24h 内你可否决任一分身决议              │
│                                                 │
│ [一键授权] [自定义边界] [取消, 我自己来]        │
└─────────────────────────────────────────────────┘
```

### 3.3 会议进行中 (员工视角)

员工手机/Web 收到**实时简讯** (非紧急可静默):

```
13:45  📨 你的分身已加入"Q2 用户增长策略评审"
14:08  🗣 分身已汇报 KR-001 (基于你的 Check-in)
14:15  ⚠️ 议题超出授权: "是否砍掉 X 功能"
       → 已暂停发言, 请你 5 分钟内决断
       [批准砍掉] [反对砍掉] [让分身投弃权]
14:22  ✅ 决议产出 1 张: "X 功能延期至 Q3"
14:30  📋 会议结束, 摘要已生成 → [查看]
```

**关键铁律**:

- 分身在超出授权时**不强行决策**, 实时弹给员工
- 员工 5 分钟未响应 → 默认弃权
- 紧急中断会议时, 分身可主动告知"我请示员工后回复"

### 3.4 会后否决窗口 (24 小时)

```
┌─────────────────────────────────────────────────┐
│ 📋 你的分身今日代参 1 场会议 (28 分钟)          │
│                                                 │
│ 产出 Decision Cards:                            │
│ ✅ DC-7901 KR-001 进度承认 (+8%)               │
│ ✅ DC-7902 X 功能延期至 Q3 (你已批准)          │
│ ⚠️ DC-7903 同意月度复盘改为双周 (待你 review)   │
│                                                 │
│ 你的发言摘要 (基于你历史立场风格生成):          │
│ • 强调 KR-001 推进速度优于预期                  │
│ • 提议邀请客户成功团队下次参会                  │
│                                                 │
│ ─────────────────────────────────────────────  │
│ ⏰ 否决窗口: 23h 56m                            │
│ [一键确认] [否决某条] [回看会议片段]            │
└─────────────────────────────────────────────────┘
```

否决机制:

- 24h 窗口期内员工可否决任意 Decision Card
- 否决 → 自动通知会议参与方"该决议被员工修正"
- 会议参与方可重新讨论或接受修正
- 否决记录沉淀到员工的拿捏老板成长档案

### 3.5 控制中枢: 跨 SKU 统一

**重要原则**: 事后纪要确认对所有 SKU 开放, 不基于商业等级 gate.

| SKU | 分身代参 | 音色 | 否决窗口 |
|---|---|---|---|
| Free | ✅ 开放 | AI 通用音色 | 24h |
| Personal | ✅ 开放 | AI 通用音色 | 24h |
| Enterprise | ✅ 开放 | 企业可采集本人音色 | 24h |

**为什么不 SKU gate**:

- 这是 PLG 杀器, gate 会损失病毒传播
- 控制点是事后确认, 不是准入
- 员工尊严不因付费等级有差别

## 第四章: 红黄绿三档授权机制

### 4.1 三档定义

```
┌─────────────────────────────────────────────────┐
│         会议级别 → 分身参会权限                  │
├─────────────────────────────────────────────────┤
│                                                 │
│  🟢 绿区: 内部日常会议                          │
│  ─────────────────────────                     │
│  • Standup / 周会 / 复盘                       │
│  • 跨团队同步会                                 │
│  • 信息同步会                                   │
│  → 分身可自动代参 (默认开)                      │
│  → 一键授权即可                                 │
│                                                 │
│  🟡 黄区: 重要内部会议                          │
│  ─────────────────────────                     │
│  • Q 度评审 / 大型项目 Kickoff                 │
│  • 涉及多 KR 调整                               │
│  • 跨部门资源协调                               │
│  → 分身可代参, 但每次需员工事前授权             │
│  → 关键议题强制弹回员工决策                     │
│                                                 │
│  🔴 红区: 必须本人会议                          │
│  ─────────────────────────                     │
│  • 客户 / 合作伙伴 / 供应商会议                │
│  • 招聘面试 / 绩效面谈                          │
│  • 战略 / C-level / 董事会                      │
│  • 法务 / 合规 / 审计 / 财务                    │
│  → 分身被禁用                                   │
│  → 系统强制只能选"本人参加 / 拒绝"             │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 4.2 红区清单配置

**配置主体**: 公司管理员 + 主持人**双配置**

```
公司管理员配置 (全局):
- 包含外部参会人 (邮箱不在企业域) → 红区
- 议程含"客户/合同/法务"等关键词 → 红区
- 标记为机密的会议室 → 红区
- 由 HR/法务/财务部门发起 → 红区

主持人配置 (本次):
- "仅本人参会"开关 → 强制红区
- 拒绝某员工分身参会 → 仅对该员工红区

冲突处理:
- 公司红区 + 主持人许可 → 仍红区 (公司优先)
- 公司绿区 + 主持人禁止 → 红区 (主持人有权升级)
```

### 4.3 跨公司会议

跨公司会议 (含外部邮箱) 默认红区. 解锁条件:

- 双方都使用牛马搭子 + 双方书面同意
- 双倍水印强制 (双方分身都标识)
- 红区清单中明确加入

## 第五章: 分身能力分级 (与拿捏老板挂钩)

不是所有员工的分身都能立即代参. 与拿捏老板成长阶段挂钩:

| 成长阶段 | 分身参会能力 | 可代参场景 |
|---|---|---|
| Lv.1 🥚 新手 (0-2w) | ❌ 不能代参 | 仅旁听记录 |
| Lv.2 🐣 上手 (2w-2m) | ⚠️ 仅 standup | 只能汇报数据, 不能表态 |
| Lv.3 🐤 熟手 (2m-6m) | ✅ 绿区会议 | 可表态 + 简单承诺 |
| Lv.4 🦅 老手 (6m-1y) | ✅ 黄区会议 | 可承诺 ≤ 1 工作日动作 |
| Lv.5 🐉 拿手 (> 1y) | ✅ 跨企业 | 可参加跨企业 (除红区外) |

**意图**: 这一设计创造了员工"养分身"的真实动力——分身越成熟, 能替你做的事越多.

## 第六章: 现场切换 (无缝接管)

像微信通话中可"切到文字"一样, 会议进行中也能切换:

### 6.1 场景 A: 分身参会中, 员工突然有空

```
员工点 [我加入]:
  • 会议室提示: "李四本人已加入, 分身退出"
  • 分身把已发言摘要私聊给员工
  • 员工无缝接管 (戏剧时刻!)
```

### 6.2 场景 B: 员工参会中, 临时有急事

```
员工点 [让分身接管]:
  • 会议室提示: "李四临时离开, 分身代参"
  • 员工把"我已说过 X, 立场是 Y"快速交代
  • 分身接力, 维持已有立场
```

### 6.3 场景 C: 双轨并存 (伴随模式)

```
员工开 [伴随模式]:
  • 员工本人发言时, 分身待机
  • 员工放空时, 分身在屏幕侧栏给提示:
    "现在该你回应了, 你 Q1 的立场是..."
  • 类似"耳麦提词器"
```

伴随模式仅企业版开放 (需采集员工历史发言风格).

## 第七章: 多人分身参会 (会议必要性市场检验)

```
┌─────────────────────────────────────────────────┐
│   "这场会需要本人吗?" 自然市场检验               │
├─────────────────────────────────────────────────┤
│                                                 │
│  分身参与率   含义              系统响应         │
│  ──────────  ────────────────  ──────────────  │
│   < 30%      正常会议           按常规进行      │
│   30-70%     部分人能代          继续, 但提示   │
│   > 70%      这会本来就该不开!   ⚠️ 系统弹窗:   │
│                                  "本会 5/6 人选 │
│                                  分身, 是否改   │
│                                  为异步会议?"   │
│   100%       AI 之间的会议       ⚡ 压缩为 2 min│

```

**深层意义**: 牛马搭子用"分身代参率"作为**会议必要性的市场指标**. 率高就建议异步化. 慢慢杀死"为开会而开会"的中国职场顽疾.

## 第八章: 法律 / 合规边界

| 风险 | 处理 |
|---|---|
| 分身承诺合同是否有效 | 仅在内部会议有效; 对外承诺需员工本人事后确认签字 |
| 分身参加纪检 / 法务会议 | 默认禁止; 不可解锁 |
| 分身参加涉密 / 涉及国家机密 | 必须本人 + 不可录制; 技术上检测关键词阻断 |
| 分身代签客户/合作伙伴会议 | 红区强制; 除非客户也用牛马搭子且双方协议 |
| 公司是否要公告 "AI 代参政策" | ✅ 必须; 员工手册章节 + 会议邀请页注明 |
| 会议主持人能否拒绝分身参会 | ✅ 必须能; 主持人有"仅本人参会"开关 |
| 跨国 / GDPR / 涉外 | V1 不支持; V2 单独合规版本 |
| 分身参会被识破当事人不知情 | ✅ 强制声明: 分身入会自动加水印 "AI 代理" |

**最重要**: 所有分身入会**必须有显式标识**, 不能"假装是真人". 否则违反诚信原则 + 可能触犯反欺诈法.

## 第九章: 技术实现路径

### 9.1 V1: 文字代参 (3-6 个月)

```
• 调腾讯会议 OpenAPI 拉录制 + 字幕
• 分身不进会议室, 但实时分析直播流
• 关键时刻通过会议聊天框 (chat) 发言:
  "[李四的分身]: 关于 KR-001..."
• 会后生成完整 Decision Card

技术风险: 低 (纯文本)
商业价值: 已经能完成 80% 杀手叙事
```

### 9.2 V2: 语音代参 (6-12 个月)

```
• 分身以"语音参会"加入会议室
• ASR + LLM + TTS (员工本人音色)
• 实时对话延迟 < 2 秒
• 静态头像 + 唇形同步可选

技术风险: 中 (实时性挑战)
商业价值: 杀手 feature 完整版
```

### 9.3 V3: 视频 Avatar 代参 (12+ 个月)

```
• 3D / 数字人 Avatar (员工形象)
• 表情 / 手势同步

技术风险: 高 (CG + 实时渲染)
商业必要性: 待验证 (V2 可能已够)
```

**节奏建议**: V1 文字代参就足够打动客户; V2 语音代参作为正式杀手 feature 推广; V3 视频可选, 创业期不必早做.

## 第十章: 腾讯会议接入策略

### 10.1 大客户路径

```
客户已有腾讯会议企业版授权
  ↓
牛马搭子作为应用接入腾讯会议 OpenAPI
  ↓
单租户 OAuth 授权
  ↓
开箱即用, 无配置成本
```

### 10.2 小客户路径

```

  ↓

## 第十一章: 反例与禁忌

### 11.1 禁止的产品形态

| 反例 | 为何禁止 |
|---|---|
| 静默代参 (对方不知是 AI) | 反诚信, 违反第九条 |
| 一次授权终身代参 | 失控, 员工实际无控制 |
| 红区会议白名单加入例外 | 红区是合规底线 |
| 代参可关水印 | 违反显式标识铁律 |
| 主持人不能拒绝分身 | 主持人有权 |
| 强制企业采集员工音色 | 违反员工尊严 |
| 用 SKU gate 否决窗口 | 员工尊严不分等级 |
| 分身代员工被骂 | 违反第十三条 13.2 |

### 11.2 文化反例

| 反例行为 | 应对 |
|---|---|
| 员工总是分身代参 (从不本人) | 拿捏老板提示"本月 100% 代参, 关注会议必要性" |
| 经理强制员工本人参会 (非红区) | 治理官介入审视 |
| 利用分身参会逃避问责 | 否决窗口 + 责任追溯 |
| 分身代参后员工不 review | 拿捏老板提示, 多次未 review 自动降级权限 |

## 第十二章: 数据指标 (产品健康度)

```
代参核心指标:
  • 代参覆盖率: 目标 > 30% 会议时长
  • 代参满意度 (员工 NPS): > 50

  • 红区误判率: < 1%
  • 分身被识破争议: 0 (强制水印应消除)

效能指标:
  • 员工心流时间增加: 平均 +1h/天
  • 8h 外会议参与下降: 60-80%
  • 会议满意度提升: +20pp

## 结语: 重新定义"在场"

工业时代的"在场" = 物理身体出现在会议室
互联网时代的"在场" = 视频窗口打开
**AI 时代的"在场" = 你的意志被准确代表**

牛马搭子的分身代参, 不是"让员工偷懒", 是**让员工的意志可以被准确委托**.

> **「会议的本质不是出席, 是决议. 牛马搭子让决议归你, 出席自由.」**

## 附录 B: 用户教育 (新员工 5 分钟培训)

```
1. 收到会议邀请, 看 3 个选项:
   • 👤 本人参加 (默认)
   • 🤖 分身代参 (你忙时)
   • ❌ 拒绝 (说明理由)

2. 选分身代参:
   • 看授权卡, 一键授权或自定义边界
   • 红区会议 (客户/合规等) 不能选分身

3. 会议中:
   • 收到实时简讯, 重要节点会弹回你决断
   • 5 分钟未响应 → 默认弃权

4. 会议后:
   • 看 Decision Card 摘要 + 你的发言摘要
   • 24h 内可否决任意条
   • 一键确认完成

5. 切换:
   • 分身代参中, 你有空 → [我加入]
   • 你参会中, 有急事 → [让分身接管]
```
