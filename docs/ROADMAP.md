## 摘要 (TL;DR)

```
V1 MVP    → 12-14 月 (60+ 功能, OSS 借力 + 自建差异化) → 完整产品 GA
```

**关键认识**:

- **OSS 借力**: IM/邮件/日历/文档/云盘 全部 fork 开源底座 (16 人月, 不重造轮子)
- **AI 加速**: Cascade/Cursor 加持下, 实际产能 3x 传统团队
- **思考层自建**: TAF/OKR/议事室/拿捏老板/知识架构 100% 投入 (37 人月)
- **战略锁定**: 见 `MANIFESTO.md` 第十八条 + `OSS-STACK.md`
- **预算**: 12 人 × 14 月 ≈ 1200 万 RMB

### 1.2 关键判断

**好消息**:
- Hermes 已有 OKR 模块基础, 类别一/二/三 大部分可复用
- React + Tailwind UI 框架成熟, 视觉迭代快
- AI 能力依赖 LLM API, 无需自训模型 (V1)

**挑战**:
- 议事室 + 3+1 + 拿捏老板 是真正的 0-1 创新, 需打磨
- 腾讯会议 OAuth + 实时流处理是新栈
- 双轨 KPI/TTI 需重写绩效模块

**选择**:
- **不重写 Hermes**, 在其上演进
- **优先创新模块** (V1 必含 6 个独创)
- **集成生态延后** (V2 再做钉企微飞书)
