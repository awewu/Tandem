# 拿捏 OKR 完整体验架构

> 配套文档：`PRD.md`（系统全局）/ `NIP-SPEC.md`（IM 协议，待写）
> 版本：v0.1（2026-05-07）
> 目的：把 OKR 从"一个模块"升级为**贯穿拿捏整个系统的工作主轴**，并定义完整端到端体验。

---

## 1. 设计哲学：OKR 是操作系统的"内核中断"

> **当一件事发生（聊天、写代码、提交审批、开会），系统都问一句：这件事跟哪条 OKR 相关？**

把 OKR 从一个被动填表的页面，变成**主动响应工作行为的反射层**：

| 维度 | 传统 OKR (Tita/飞书) | 拿捏 OKR |
|---|---|---|
| **定位** | 季度填一次的表格 | 系统级"工作意图"中枢 |
| **数据** | 人手工填 Check-in | AI 从行为流自动汇总 |
| **可见** | 进 OKR 页面才能看 | 群聊/任务/邮件每个角落都能看到关联 OKR 卡 |
| **对齐** | 上下级二维树 | 多维：层级 + 主题 + 跨部门协作网 |
| **节奏** | 周/双周提醒 | 自适应（节奏脱漏 → 自动告警 → 议事室 → 修复） |
| **真相** | 自评打分（容易粉饰） | 行为数据交叉验证 + Baseline Drift 检测 |
| **复盘** | 季末写复盘文档 | 每周生成微复盘 + 季末聚合 |

### 1.1 五大体验理念

| 理念 | 含义 | 关键体现 |
|---|---|---|
| **牵引 (Alignment)** | OKR 牵引每天的工作选择 | 每日工作台首屏看 OKR；任务/会议默认要选关联 KR |
| **透明 (Transparency)** | 默认公开，让协作浮现 | 全公司 OKR 树可看；每个 KR 关联群聊全员可观察 |
| **节奏 (Rhythm)** | 节拍可感知可干预 | Cadence 徽标 + 议事室 + 健康度告警三件套 |
| **真相 (Truth)** | 反对粉饰，鼓励诚实 | 0.7 高分文化 + Drift 检测 + 自动数据交叉 |
| **进化 (Evolution)** | OKR 模板与方法论持续学习 | 复盘沉淀 → 模板库自动积累 → 新周期推荐 |

## 2. OKR 完整生命周期（五阶段）

```
   ┌──────────────────────────────────────────────────────────────┐
   │                                                              │
   ▼                                                              │
┌───────┐   ┌───────┐   ┌───────┐   ┌───────┐   ┌───────┐         │
│ 制定  │ → │ 对齐  │ → │ 执行  │ → │ 复盘  │ → │ 进化  │ ────────┘
│Draft  │   │Align  │   │Execute│   │Review │   │Evolve │
└───────┘   └───────┘   └───────┘   └───────┘   └───────┘
   1 周        1 周         12 周        2 周         持续
```

### 2.1 阶段一 · 制定 (Drafting)

**目标**：把模糊的方向变成 SMART 的 O+KR+Initiative。

**用户旅程**：
1. 周期开启自动通知 → 工作台首屏出现"新周期 OKR 待制定"
2. 一键 **AI Draft**：分身基于历史 OKR、上级 OKR、近 90 天的会议/邮件/Git 数据 → **生成 3 套候选**
3. 模板库选场景模板（销售/产品/工程/HR 等 11 套已内置）
4. 拖拽编辑、写描述
5. **质量检查器** 实时打分：SMART 满足度 / O 是否模糊 / KR 是否可量化 / 数量是否 ≤ 5
6. 提交 → 进对齐阶段

**AI 介入**：
- **Draft Agent**：从员工历史输出推断"自然延续"的 KR
- **Quality Linter**：实时高亮"O 含数字"、"KR 没有可量化指标"等反模式
- **Baseline 守门**：如果员工写的 KR 违反公司价值观（如"不计成本提升 X"），标红警告

**产出**：草稿状态的 Objective + KR + Initiative 候选

### 2.2 阶段二 · 对齐 (Alignment)

**目标**：上下级、横向同侪在 OKR 矩阵上达成共识。

**用户旅程**：
1. 草稿提交 → 自动通知直属上级 + 关键依赖人
2. 经理在 **「对齐视图」**（树形 + 跨部门连线图）看下属 OKR 全景
3. **多分身议事室**：经理 @ 5 名下属的 AI 分身，议事室里分身代表员工"陈述目标合理性"，经理只听结论
4. **横向对齐**：员工 A 的 KR 依赖员工 B 的能力 → 系统自动建议"链接 B 的某 KR 作为 dependency"
5. 1:1 对齐会：日历自动安排，会前 AI 准备议程（双方草稿对比 + 历史成败模式）
6. 经理审批 → 进入执行

**AI 介入**：
- **Alignment Coordinator**：自动识别可能冲突（两个员工目标值矛盾）/ 重复（多人 KR 雷同）/ 缺位（部门战略未被任何下属承接）
- **依赖图自动构建**：从 KR 文本抽取关键人/系统，提示"这个 KR 依赖谁"
- **议事室纪要**：自动产出"对齐结论 + 待修改项 + 风险项"

### 2.3 阶段三 · 执行 (Execution)

> 这是 12 周的主战场，**所有差异化体验集中在这里**。

#### 2.3.1 每日工作台 (Daily Dashboard)

```
┌─ 早 09:00 我的工作台 ─────────────────────────────────────┐
│ 你的 3 个 KR：                                            │
│ ◉ 留存 38%/45%       ⏰ 2天内 Check-in   🟢 信心：on     │
│ ◉ 推荐覆盖 80%/100%  📅 4天后             🟡 信心：风险  │
│ ◉ 文档完成 60%/100%  ⚠ 逆期 3天          🔴 信心：偏离  │
│                                                           │
│ 今日建议（AI 分身生成）：                                 │
│ • 上午：跟进新手引导 A/B 实验（关联 KR1）                 │
│ • 下午：处理推荐覆盖率阻塞——已发消息给基础架构组         │
│                                                           │
│ 待审：3 条 AI 自动起草的 Check-in →                       │
└───────────────────────────────────────────────────────────┘
```

#### 2.3.2 OKR 原生群聊（杀手级）

每个 Objective 自动创建一个群（`Conversation(type=okr_group)`），所有 KR 讨论汇聚此处：

- 群成员 = Owner + 协作者 + 关注者 + Owner 的 AI 分身（🤖 水印）
- 发消息可以挂"OKR 卡"引用某个 KR
- 长按消息 → "转为 Check-in" 一键提交进度
- 长按消息 → "转为行动项" 直接生成 Initiative
- 群里发数据 → AI 自动识别 → 浮出"是否更新 KR3 当前值为 82%？" 卡片
- 整个 OKR 季度的所有讨论都沉淀在这一个群里，**复盘时不再考古**

#### 2.3.3 自动 Check-in（不再周报）

```
每天 18:00 / 周五 17:00
   ↓
AI 扫描：聊天 + 邮件 + Git + Jira + 文档编辑
   ↓
生成 Check-in 草稿：
  • KR1 留存：38% (+2)  来源：周二群聊讨论 + B/I 数据
  • KR2 推荐：80% (+0)  无明显进展
  • 障碍：UX 招聘卡住    来源：HR 邮件
  • 下周计划：完成新手引导上线 + 启动 A/B
   ↓
推送到员工"待审"区
   ↓
员工 30 秒确认 → 一键发布 → OKR 群可见
```

#### 2.3.4 议事室（多分身议事）

经理在 OKR 卡片右键 → "组织议事室"：
- 5 个 KR Owner 的分身轮流陈述本周进展、阻塞、需协调事项
- Coordinator Agent 自动总结："建议本周聚焦新手引导，UX 招聘升级到 HRD"
- 流式同步给经理观察，经理可随时插话/打断
- 议事结束自动产出：纪要 + 待办行动项 + KR 状态变更草稿

#### 2.3.5 节奏与告警

- **Cadence 徽标**（已实现）：`📅 5天 / ⏰ 2天内 / ⚠ 逆期 N天`
- **健康度规则**（已实现）：停滞 / 信心连降 / 进度倒退 / 权重失衡
- **自动升级**：逾期 3 天 → 推送 Owner；7 天 → 推经理；14 天 → 部门负责人
- **熔断式停摆**：信心连续 3 周 off-track + 进度倒退 → 系统强制弹出"是否调整 KR？"

#### 2.3.6 1:1 教练模式

经理与下属 1:1 时，系统自动生成议程：
- 上次 1:1 提及但未 follow up 的事项
- 该下属本周 OKR 进展异常项
- 同侪同 KR 的最佳实践
- AI 推荐的 3 个开放性问题（GROW 框架）

### 2.4 阶段四 · 复盘 (Review)

**目标**：诚实评估、提取教训、为下一周期蓄能。

**用户旅程**：
1. 周期最后一周自动启动复盘流程
2. **自评**：员工用 0.0–1.0 滑块，**AI 给建议值**（基于实际数据）
3. **同侪互评**：协作者匿名评 1 项："对方在哪个 KR 上对你帮助最大/最少"
4. **上级评分**：经理参考自评 + 同侪 + 数据
5. **终评**：HR / 部门头确认（防止部门内通胀）
6. **复盘文档** 自动生成（PDCA / KISS / 4L 三选一），员工补充 2-3 段反思
7. **公开复盘会**：自愿，每周期挑 1-2 个标杆案例分享

**AI 介入**：
- **Score Recommender**：交叉数据 + Drift 报告 → 给出建议分
- **Pattern Extractor**：识别"承诺过满"、"季中漂移"、"协作过度依赖某人"等反复出现的模式
- **Insight Card**：生成员工个人能力成长画像

### 2.5 阶段五 · 进化 (Evolution)

**目标**：把这一季学到的方法论沉淀为下一季的起跑点。

**机制**：
1. **模板库自动学习**：系统从该季所有"高分 + 完成度高 + 健康度好"的 OKR 抽取共性 → 候选模板 → HR 审核入库
2. **经理素材库**：每个经理的"对齐金句"、"激励话术"、"难题处理范式"沉淀为该经理的 Persona Memory
3. **公司级 Insight 月报**：CEO 看板呈现"哪类 OKR 完成率最高"、"哪个部门 Drift 最频繁"等元层洞察
4. **Baseline 演化**：本季暴露的红线漏洞自动建议补充进 Company Baseline

## 3. 五种角色的差异化体验

### 3.1 员工 (IC) — 减少汇报、聚焦输出

| 痛点 | 拿捏方案 |
|---|---|
| 写日报浪费时间 | AI 自动汇总每日产出 → 一键发布 |
| 不知道在干啥才是对的 | 每日工作台首屏看到 KR 优先级 |
| 1:1 不知道说啥 | AI 准备议程 |
| OKR 进度更新麻烦 | Auto-Updater 起草，员工 30 秒确认 |
| 复盘写不出来 | AI 起草 + 同侪互评 |

**核心 KPI**：每周节省汇报时间 ≥60 分钟；周 Check-in 达成率 ≥80%。

| 1:1 准备 | 系统自动议程 |
| 议事高效 | 多分身议事室 |
| 跨部门协调 | 横向 OKR 依赖图 |

### 3.3 部门负责人 (Director) — 战略到执行的翻译器

| 需求 | 拿捏方案 |
|---|---|
| 战略落到每个员工 | 对齐视图 + 缺位检测（哪条战略没人承接）|

## 4. OKR 作为信息架构的中枢

> OKR 不在自己的页面里**孤立存在**，而是**贯穿其他所有模块**。

### 4.1 与 IM 整合

| 触点 | 体验 |
|---|---|
| 群聊 | 每个 OKR 自动有群；群里挂 OKR 卡；消息可转 Check-in |
| 1:1 私聊 | AI 分身自动私发 Check-in 草稿；1:1 议程含 OKR 上下文 |
| 频道/广播 | CEO 周报自动广播；公司大目标公开变更日志 |
| @通知 | @某 KR Owner = @这个人 + 引用该 KR 卡片 |

### 4.2 与任务/工作台整合

- 创建任务时**默认要求**选择关联 KR（可选"无关联"但需理由）
- KR 看板可下钻到 Initiative → Initiative 下钻到 Task
- 任务完成事件自动尝试更新 KR 进度
- 任务"星标"= 该 KR 优先

### 4.3 与日历/会议整合

- 周期开启自动安排：制定 → 对齐 1:1 → 周会 → 月中 review → 季末复盘
- 任何会议可关联 1 个或多个 KR；会议纪要自动归档到 OKR 群
- 议事室预约一键生成

### 4.4 与文档整合

- 协同文档可挂 OKR 卡作为"目标说明"
- 文档编辑事件被 OKR Auto-Updater 监听
- 复盘文档模板内置（PDCA/KISS/4L）

### 4.5 与审批/邮件整合

- 报销/出差审批可声明"为达成 KR X"，便于事后归因
- 邮件订阅"我的 OKR 周报"

### 4.6 与考勤整合（可选）

- 不直接挂钩，但 HR 看板可看"加班时长 vs OKR 完成"是否健康（防止内卷）

## 7. 数据架构（OKR 在系统中的"骨骼")

### 7.1 实体扩展（基于 V0 已有）

```sql
-- 已有
Cycle, Objective, KeyResult, Initiative, CheckIn,
Comment, Activity, Score, Watcher, Retrospective

-- 新增（多租户 + 集成）
Cycle.cadence_pattern        -- 周/双周/月 + 自定义节奏点
Cycle.review_template        -- PDCA/KISS/4L
Objective.dependencies[]     -- 跨人依赖图
Objective.conversation_id    -- 关联 OKR 群
Objective.calendar_events[]  -- 关联会议
Objective.draft_metadata     -- AI 起草来源 + Baseline 版本

KeyResult.data_sources[]     -- 数据来源（自动接 BI/CRM/Jira）
KeyResult.auto_update_rules  -- 抽取规则 YAML
KeyResult.last_extracted_at  -- AI 上次抽取时间

CheckIn.source_type          -- manual / auto / mixed
CheckIn.evidence_refs[]      -- 来源消息/PR/邮件 ID
CheckIn.persona_drafted      -- 是否分身起草

Score.recommended_value      -- AI 建议分
Score.drift_score            -- 真相分（基于行为数据）

Retrospective.template_type  -- PDCA / KISS / 4L
Retrospective.ai_drafted     -- 是否 AI 起草
Retrospective.shared_publicly-- 是否对外分享为标杆
```

### 7.2 跨模块外键

- `Conversation.related_okr_id` → Objective

- `Email.tagged_okr_ids[]` → Objective

## 9. 成功度量（北极星与辅助）

### 9.1 北极星指标

> **每位活跃员工每周节省汇报时间 ≥60 分钟，且周 Check-in 达成率 ≥80%**

### 9.2 辅助指标

| 维度 | 指标 | 目标值 |
|---|---|---|
| **采纳** | 月活员工 / 总员工 | ≥85% |
| **效率** | AI 草稿被一键采纳率 | ≥70% |
| **质量** | OKR 通过质量检查率 | ≥80% |
| **真相** | Drift 分均值 | ≤0.15 |
| **节奏** | 周 Check-in 达成率 | ≥80% |
| **协作** | 平均跨人依赖数 / OKR | 1.5–3 |
| **复盘** | 复盘完成率 | ≥90% |
| **沉淀** | 标杆案例 / 季 / 100 员工 | ≥3 |
| **健康** | 健康度告警从触发到处理 P50 时长 | ≤2 天 |

### 9.3 反指标（要警惕）

| 现象 | 含义 |
|---|---|
| 自评分均值 > 0.85 | 文化通胀；推 0.7 高分文化教育 |
| 0.0–1.0 分布双峰 | 部分粉饰部分摆烂 |
| KR 数 > 5 / Objective | 焦点散；推质量检查 |
| Check-in 全靠 AI 草稿无修改 | 员工"机械确认"；改进引导 |
| OKR 群消息密度 < 5 条/周 | OKR 与日常脱节 |

## 10. 风险与对策

| 风险 | 影响 | 对策 |
|---|---|---|
| 员工不信 AI 替自己写 Check-in | 推不动 | L1 默认；草稿可见来源；员工随时撤回 |
| AI 抽取错把闲聊当进展 | 信任崩 | 抽取必须有"来源链接"；Drift 检测兜底 |
| 经理用议事室代替真 1:1 | 关系疏远 | 1:1 议程仍在；议事室不能完全替代 |
| 公开 OKR 暴露隐私 | 抗拒 | 三档可见性：私有/部门/全员 |
| 复盘流于形式 | 知识不沉淀 | AI 强制要求填 2 段反思才能完成 |
| 模板库越来越大无人维护 | 失效 | HR 季度审核；自动淘汰 12 月未使用 |
