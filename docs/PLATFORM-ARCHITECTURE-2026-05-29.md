# Tandem Platform Architecture — 2026-05-29

> 本文是 2026-05-29 设计会的全量入档：18 项架构决议 + 4 条宪章护栏 + AI 三层架构 + 信息架构边界 + 实施路线。

## 0. 名词与原则

- **Tandem**：原"搭子"。员工的个人 AI 工作台，1 主舞台 + 2 召唤侧栏。**不使用 buddy / sidekick 等英文别名**。
- **事半**：OKR 驱动的工作执行域（决策卡 / 议事室 / 复盘 / 看板）。
- **议事室**：17 分钟收敛的决策会场（宪章 §5）。
- **决策卡**：原子级的"做不做 / 怎么做"载体（宪章 §1）。
- **AI 三层**：通用 AI（GPT/Kimi）、中央 AI（公司大脑）、个人主分身（员工 AI）。
- **资产 4 级 ownership**：公司 / 部门 / 团队 / 个人。

**贯穿原则**：
1. 舞台 ≥ 60% 屏宽，侧栏召唤即用、不召即收。
2. 跳板（Launchpad）位置不可动；跳转日志、AI 推荐、品牌砖墙保留。
3. AI 永远是"3+1 选项"，不替员工拍板（宪章 §2）。
4. 关键投票必须真人确认（本文新护栏 §G1）。

## 1. 18 项架构决议

### A. 命名与主结构

**D1. 搭子 → Tandem**
- 全平台 UI / 文案 / 路由统一 `Tandem` / `/tandem`。
- 保留中文副标"个人工作台"，不再出现"搭子 / Buddy"。

**D2. Tandem 主结构：1 舞台 + 2 召唤**
- 主舞台 = 当前任务 / 决策卡 / 文档（占屏 ≥ 60%）。
- 左召唤：**身份栏**（我的分身、Memory、技能、形象、成长）。
- 右召唤：**行动栏**（待办、议事、IM 未读、AI 推荐）。
- 召唤侧栏默认折叠为图标条，宽度 280px，可锁定。
- 拒绝三栏常驻拥挤布局。

**D3. 首页 ≠ Tandem**
- 首页 `/` = **风险揭示型 Dashboard**（公司 / 部门 / 个人三层风险卡）+ Launchpad 跳板。
- Tandem `/tandem` = 个人工作台。
- 现首页的"我的视角"组件全部迁出到 `/tandem`，首页只留"全局风险 + 跳板"。

**D4. Launchpad 跳板原位保留**
- 位置：首页右侧 1/3。
- 行为：点击日志、AI 智能排序、品牌砖墙、外链跳转全部保留。
- 任何重构禁止动 Launchpad 接口和数据流。

### B. 事半与 Tandem 的边界

**D5. Tandem ↔ 事半 双向唤起**
- 从 Tandem 主舞台点决策卡 → 跳 `/decisions/[id]`。
- 议事室结束 → 回写 Tandem 右召唤栏"我的待办 / 待复盘"。
- 复盘产出 → 沉淀到 Tandem 左召唤栏"我的 Memory / 代表作"。

**D6. 报告与汇报留在事半域**
- `/report` 归属事半，不放进 Tandem。
- Tandem 仅提供"起草入口"按钮唤起 `/report/new`。

### C. IM 模块（企微替代）

**D7. IM 三栏定位不变**
- 频道列表 / 消息流 / 频道详情。
- 已具备：议事室升级、@分身召唤、消息升 Memory、置顶、@群公告、紧急未读。

**D8. IM 待补能力（P1）**
- **群待办**：消息可一键转为待办，挂到 `/tandem` 右召唤栏。
- **群文件**：频道内 Drive 视图，文件归属 = 该频道部门 / 团队。
- **群机器人**：议事室定时提醒、OKR 进度播报。
- **消息引用 + 跨频道转发**。

**D9. 议事室在 IM 内启动，但跳出到 `/decisions/[id]`**
- IM 消息悬浮 → "开议事室"按钮已存在，保留。
- 议事室结果回写 IM 系统消息（已实现），保留。

### D. 资产矩阵（4 级 ownership × 3 类资产）

**D10. 统一资产矩阵**

|              | 公司           | 部门               | 团队           | 个人              |
| ------------ | -------------- | ------------------ | -------------- | ----------------- |
| **Memory**   | 公司 Memory    | 部门 Memory ⚠缺UI | 团队 Memory ⚠ | 个人 Memory       |
| **文件**     | 公司盘 ⚠缺UI  | 部门盘 ⚠缺UI      | 团队盘 ⚠      | 个人盘（现 Drive） |
| **文档协作** | 公司空间 ⚠    | 部门空间 ⚠         | 团队空间 ⚠     | 个人 Documents    |

- 后端 `MemoryOwnershipLevel` 已支持 4 级。
- 前端 P1 补：Memory / Drive / Documents 三处都加部门 + 团队 tab。

**D11. 邮箱补完**
- 现状：仅 SMTP 发，无 IMAP 收。
- P1：IMAP 收信 + 多 folder + AI 起草 + AI 摘要。
- 设置入口 `/settings/email` 已存在，保留。

**D12. 个人产出库（Portfolio）实装**
- 现 `/portfolio` 是占位。
- P1：
  - 自动聚合日报 / 复盘 / 议事发言 / Memory / 创意。
  - 手动加精（"标星代表作"）。
  - 同事背书（≠ 点赞，需写一句话）。
  - 离职导出 PDF + 加密包，归属员工本人。

### E. AI 三层架构

**D13. 三层定位**

| 层      | 名字         | 入口                              | 职责                                  |
| ------- | ------------ | --------------------------------- | ------------------------------------- |
| 通用 AI | GPT/Kimi 等  | Tandem 左召唤"通用 AI 沙盒"      | 闲聊、个人成长、外部知识、不入公司库 |
| 中央 AI | Tandem Atlas | 顶栏品牌位 + 新 `/atlas` 独立栏  | 公司大脑、技能调度、监控告警、决议史   |
| 个人 AI | 我的主分身    | Tandem 主舞台 + IM @召唤          | 替我开会、起草、回邮、议事代理         |

**D14. 通用 AI 沙盒**
- Tandem 左召唤栏新增"通用 AI"tab。
- 接入第三方 LLM（GPT / Kimi / Claude / DeepSeek）。
- **不沉淀到公司 Memory**，会话仅个人可见。
- 用于个人成长、外部知识、灵感探索。
- 默认水印"非公司数据，不入档"。

**D15. 中央 AI 独立 UI：`/atlas`**
- 顶部品牌位常驻"Tandem Atlas"按钮。
- `/atlas` 独立栏包含：
  - **Skill Market**：技能市场（贡献 / 订阅 / 评分，宪章 §13 四道关）。
  - **Memory Atlas**：公司决议地图，可问答"上次 X 怎么决的"。
  - **公司之声**：CEO / 高层 AI 分身的对外稿件、月度致辞（需签批，见护栏 §G4）。
  - **监控告警**：OKR 偏差、风险线索、合规扫描。
  - **入职教师**：新人前 30 天的引导分身。
  - **Steward 副驾**：公司级决策的 AI 顾问。

**D16. 议事室 AI 与真人切换**
- 议事室成员卡片明确标识：`真人 · 王某` / `AI 分身 · 王某-Persona`。
- 切换按钮：成员可主动"切到 AI 分身代我开会"。
- **关键投票（决议级、预算级、人事级）AI 分身不可投，必须真人确认**（见护栏 §G1）。
- AI 分身发言一律打"AI 分身"水印（宪章 §15）。

### F. 治理与权限

**D17. 跳转日志全保留**
- 任何跨模块跳转走 `lib/nav/jump.ts` 统一记录。
- 重构禁止丢日志、丢 deeplink、丢 referrer。

**D18. Memory 签名链**
- 每条 Memory 入库必须有 `signedBy: userId + persona + timestamp`（宪章 §9）。
- AI 分身写的 Memory 一律标记 `byPersona: true`，UI 上不同色块。

## 2. 4 条宪章护栏（新增）

对照 `MANIFESTO.md` 后，识别 4 处潜在冲突或风险，补 4 条护栏：

### G1. 议事室关键投票必须真人

- 触发条件：议程被标记为 `decisionLevel >= 'binding'`（决议 / 预算 / 人事 / 对外承诺）。
- AI 分身只能"陈述意见"，无投票权。
- 系统在投票时弹"请真人 X 确认"，超时该票作废。
- 写回宪章：建议在 §15（AI 身份）和 §5（议事室）各加一句脚注。

### G2. 中央 AI 监控告警 ≠ 全员监控

- `/atlas` → 监控告警栏目只看：
  - OKR 进度（已公开）
  - 决策卡状态（已公开）
  - 合规风险（红线）
- **不读个人 IM 内容、不读个人邮箱、不读通用 AI 沙盒**。
- 中央 AI 拉取数据需经"数据红线网关"，越界请求自动拒绝并入审计日志。
- 写回宪章：建议补 §19（隐私）一条"中央 AI 数据访问白名单"。

### G3. 技能市场贡献署名 + 撤回权

- Skill 上架必须署名 `contributorId`。
- 贡献者可申请撤回（撤回后该 Skill 进入沉睡，不删历史调用记录）。
- 评分 / 投诉 / 复议三段式。
- 写回宪章：§13（Skill Gateway 四道关）补"署名 + 撤回 + 复议"小节。

### G4. 公司之声透明 + 签批

- `/atlas → 公司之声` 任何 AI 起草的对外稿件，发布前必须有 ≥1 名高管签批。
- 已发布稿件全员可读、可追溯草稿历史。
- 高管 AI 分身单独署名（"由王某 AI 分身起草、王某签批"）。
- 写回宪章：建议新增 §21 或在 §15 后加附则。

## 7. 最终决议（2026-05-29 锁定）

1. **中央 AI 路由 = `/atlas`**（体现“公司决议地图”隐喻，中性不抑制未来能力扩展）。
2. **通用 AI 沙盒 = 允许自建私有 LLM**（第三方 + 自建二选一可配，严格不入公司 Memory）。
3. **公司之声 = 单签高管**（默认单签，重大对外稿可临时升双签）。
4. **Portfolio 离职导出 = HR 审核后导出**（员工提交 → HR 审核 → 签名 PDF + 加密包）。
