## 三、6 条立项初心 (Owner 2026-05-27 PT 22:25 原话)

> ⚠️ 本节 6 条是 Tandem 企业级 Agent 的**根本目的**, 不可在没有 Owner 重新定义的情况下被改写.

### 1️⃣ 企业级 AI 和个人 AI 的核心区别 = 组织目标和战略的聚焦达成

- **个人 AI** 解的是 "个人聪明" (个人 IQ 放大器)
- **企业级 AI** 解的是 "**组织目标达成 + 战略聚焦**"
- 这不是程度差异, 是**目的差异**

### 2️⃣ 企业级 AI 的进化方向 = 帮助整体组织能力提升 + 约束组织内工作聚焦

- 不是"AI 自己变聪明"
- 而是双向能力:
  - 🟢 **能力提升**: 员工 + AI + 流程整体变强
  - 🔴 **聚焦约束**: 防止员工/AI 跑偏 OKR

### 3️⃣ 兼容开放个人 AI 智能成长 + 个人 AI 反哺企业级 AI 成长

- 员工可以**自由用市面上所有智能体技能** (Claude Code / Cursor / OpenClaw / Hermes Agent / ChatGPT 等)
- 同时, 个人 AI 的产出**反哺**企业级 AI 的成长
- **Tandem 不重发明个人 AI, Tandem 做产出收敛器 + 网关**

### 4️⃣ 牛马 (事半) 板块 = 组织目标达成驱动器, 让大家事半功倍

- 牛马板块的**每一项**任务/自动化/通知/ToDo 都**必须**可回溯到当前 OKR
- 牛马 ≠ "工作变方便", 牛马 = "**工作聚焦在 OKR 上**"
- 跟不上 OKR 的事**不该进事半板块**

### 5️⃣ 搭子 + 拿捏板块 = 个人工作的平台和成长机制

- 拿捏 = 员工 Persona 长期养
- 搭子 = 召唤标准 Agent
- 这两个跟 OKR **解耦**, 是给员工个人成长的
- **开放接入市面所有个人 AI 智能体**, 不是封闭的 Tandem 内 Agent

### 6️⃣ 形成闭环互动, 相互赋能

- 个人 (拿捏 + 搭子) ↔ 组织 (牛马 + 企业级 Agent) 形成**双向闭环**
- 个人成长 → 反哺组织 IQ
- 组织约束 → 引导个人聚焦

## 五、Q1/Q2/Q3 精度校验结论 (Owner 2026-05-27 PT 22:40 答复)

### Q1 ✅ 事半 = OKR 驱动器, **严格版**

牛马板块的每一个任务/自动化/通知/ToDo **必须可回溯到当前 OKR**. 这意味着:

- B-005 Decision Engine 加 `okr_anchor` 字段 (必填, 不可空)
- 任何自动化触发前 verify "这跟哪个 OKR 关联"
- ProxyAction 列表里每条显示其 OKR 锚点
- 牛马板块入口看板 = OKR 进展看板, 不是 ToDo 列表

### Q2 ✅ 个人 AI 反哺 = **主动 Capture**

Tandem 主动捕获, 不靠员工手动贴:

- IDE 插件 (VSCode/Cursor extension) — 捕获 Claude Code/Cursor 产出
- 邮件 webhook — 捕获带 "Generated by ChatGPT/Claude" 标记的草稿
- 文档 metadata — Tandem 文档 schema 加 `generated_by_external_ai` 字段
- 桌面 capture (远期) — OpenClaw/Hermes 等通过 OS hook 上报

### Q3 ⭐ 接入市面智能体 = **真接入 + 中央 AI 4 道闸限制**

> Owner 原话: "可以调用但是接入后要中央 AI 给与基线和边界限制"

这不是开放也不是封闭, 是**第三种模式**: 开放接入 + 中央约束. 详见 § 六.

## 六、Skill Gateway · 中央 AI 是个人 AI 的网关

> 这是今晚最深的洞察. 它让 Tandem 产品定位**向上跃迁**: 从"组织自己的 AI" 升级为 "**个人 AI 的组织级网关**".

### 4 道闸架构

```
员工调用市面 AI 技能
(Claude Code skill / OpenClaw skill / Hermes skill / Cursor MCP / ChatGPT plugin / ...)
                    │
                    ▼
       ┌──────────────────────────────────┐
       │  Tandem Skill Gateway             │
       │  (CompanyBrain 拦截层)             │
       └────────────────┬─────────────────┘
                        │
                        ▼
       ┌────────────────────────────────────────────────┐
       │  4 道闸 (任何一道挡住即拦截或降级):                │
       │                                                │
       │  ① Baseline-Guard                              │
       │     是否违反公司 Memory                          │
       │     (跟现有 Baseline 同一套规则)                  │
       │                                                │
       │  ② OKR Drift Detection (新, 器官 #16)           │
       │     intent 是否偏离当前 OKR                      │
       │     不偏离 → PASS; 边缘 → SOFT_WARN; 远离 → 询问  │
       │                                                │
       │  ③ Data Scope (数据边界)                        │
       │     这个 skill 能读哪些数据 / 能写哪些数据         │
       │     遵守 Memory ownershipLevel + Persona scope   │
       │                                                │
       │  ④ Action Scope (行为边界)                       │
       │     这个 skill 能调哪些工具                       │
       │     重要操作进 ProxyAction 24h 否决窗            │
       │                                                │
       └────────────────┬───────────────────────────────┘
                        │ 4 闸全过 ✅
                        ▼
                 外部 skill 真的执行
                        │
                        ▼
                 产出 → Capture 层 (器官 #17)
                        │
                        ▼
                 反哺组织: Memory promotion / Skill 提议 / DecisionCard
```

### 这个网关意味着什么

| 之前的 Tandem 定位 | 校准后的 Tandem 定位 |
|---|---|
| "组织自己的 AI" (跟 Claude Code 平行) | **"个人 AI 的组织级网关"** (Claude Code 在它之下) |
| 怕员工用外部 AI (会偏离 OKR) | 鼓励员工用最强外部 AI, **网关层保证组织约束** |
| 跟 OpenAI / Anthropic 是竞品 | **OpenAI / Anthropic 是供应商**, Tandem 是企业级编排层 |
| 员工产出归员工 | 员工产出**经网关过滤后归组织**, Tandem capture 反哺 |

**最大启示**: 我们之前说 "Tandem 跟 Claude Code 不是同物种". 现在升级为: **Tandem 是 Claude Code 的"组织级老板"**. 员工用 Claude Code 写代码 OK, 但 Claude Code 调任何工具/读任何数据/改任何文件, 都要先经过 Tandem 4 道闸.

### 跟个人 agent 厂商的关系

Tandem 不跟 Claude Code/Cursor/OpenClaw 竞争, **Tandem 在它们之上**:

```
用户层    [员工]
          │
          │ 召唤
          ▼
个人层    [Claude Code] [Cursor] [OpenClaw] [Hermes Agent] [ChatGPT] [Manus]
          ↑↑↑ 员工自由选, Tandem 不替代 ↑↑↑
          │
          │ 调用 skill / 读数据 / 写文件
          ▼
组织层    [Tandem Skill Gateway]
          [CompanyBrain 4 道闸]
          ↑↑↑ Tandem 在这一层, 个人 agent 调任何东西都过这层 ↑↑↑
          │
          │ 通过后真正执行 + 产出反哺
          ▼
基础层    [文件系统] [代码库] [OKR DB] [Memory 库] [文档] [日历] [邮件]
```
