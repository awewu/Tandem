## 1. P0 修复详情

### 1.1 §4 违章 · TTI 数据库列 `yearEndBonusModifier`

**发现**: `prisma/schema.prisma:448` 还有此列.
**为什么严重**:
- TS 类型层已经删除 (`lib/types/okr-tti.ts:112` 用 `readonly affectsCompensation: false`)
- API 拒绝写 (`app/api/tti/route.ts:37`)
- 但**数据库列仍在**, 任何人通过 raw SQL / Prisma Studio / `pg_dump` 都能写入或读出
- 这是宪章 §4 "TTI 永不挂钩任何金钱回报 (含系数浮动)" 的**最后残留违章**

**修复**:

```sql
-- prisma/migrations/20260510_drop_tti_bonus_modifier/migration.sql
ALTER TABLE "TTI" DROP COLUMN "yearEndBonusModifier";
```

PRD §11 决策 [2026-05-07 删除 TTI yearEndBonusModifier 字段] **现在才真正落地** —
之前只是 TS 层声明.

### 1.2 Prisma 路径 dashboard 500

**发现**: 启动 dev server (Prisma 模式) → `GET /api/dashboard/stats` 500.

```
TypeError: (b.createdAt ?? "").localeCompare is not a function
```

**根因**: `app/api/dashboard/stats/route.ts:81` 假设 `createdAt` 是 string (符合
`DecisionCard` 类型声明), 但 Prisma `findMany()` 返回 Date 对象. 类型层和运行
时不一致.

**双层修复**:

1. **调用端防御** — `String(b.createdAt ?? '').localeCompare(...)` 兼容两种
2. **结构化** — `lib/storage/prisma-store.ts` 加 `adaptFromPrisma()`:
   - 所有 `Date` 字段自动 `.toISOString()`
   - 反向 `FIELD_RENAMES` (createdById → createdBy)
   - `get/list/create/update` 全部走该 adapter

这一步意味着 **Prisma 路径与 InMemory 路径返回的对象形状完全一致**, 业务层
(orchestrator / persona / memory flow) 不再需要知道存储后端.

## 2. 安全审计 (P1, 留待 M2)

### 2.1 API endpoint auth coverage

```
38 个 API endpoints
├── 2  AUTH-guarded (✅): /api/me/export, /api/admin/users/[id]/anonymize
├── ~6 PUBLIC by design  : /api/auth/*, /api/health, /api/integrations/health, /api/llm-health
└── ~30 OPEN (待加 guard): /api/convergence, /api/im/*, /api/memory, /api/tandem-okr,
                            /api/persona/*, /api/me/dashboard, /api/audit, /api/budget,
                            /api/cron/*, /api/realtime/*, /api/skills, /api/tandem-skills/*,
                            /api/workflows/run, /api/llm-stream, /api/dashboard/stats, ...
```

**为什么 V1 PoC 可接受**: 同机房单租户 + 自研自用阶段, 网络层物理隔离.
**M2 必须做**: `middleware.ts` 全局守门, 仅 `/api/auth/*`, `/api/health*`,
`/api/integrations/health`, `/api/llm-health` 公开, 其他全部要求 valid session.

### 2.2 `/api/me/dashboard` 接受任意 `?userId=`

```ts
// app/api/me/dashboard/route.ts:27
const userId = new URL(req.url).searchParams.get('userId') ?? 'demo-user';
```

任何人都能传 `?userId=alice@tandem.local` 看 alice 的工作台. **V1 dev 便利,
M2 必须改成从 session 拿 userId 不接受 query 覆盖**.

### 2.3 audit hash 用"简单"哈希不是 SHA-256

```
lib/audit/log.ts:hashEntry — // 简单 hash 函数 (生产期升级为 sha256)
```

链式审计在销售故事里是"数据完整性可数学验证". 用 SHA-256 才能算"密码学不可
篡改". **M6 合规阶段升级**.

+ `.eslintrc` 加 `plugins: ['@typescript-eslint']`. M2 顺手做.

### 3.2 React Hook exhaustive-deps (3 处)

```

app/im/page.tsx:161               — useEffect 缺 'loadChannels'
```
