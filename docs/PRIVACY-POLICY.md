# Tandem 隐私政策 (员工版)

> 生效日期: 2026-05-22 · 版本: v1.0
> 本政策面向使用 Tandem 的员工 (数据主体), 解释你的数据如何被收集、使用与保护.
> 你的雇主作为"数据控制者", 与 Tandem 提供方共同遵守 PIPL / GDPR / 等保二级.

---

## 1. 我们收集什么

### 1.1 你主动提供的

| 类别 | 例子 | 用途 |
|---|---|---|
| 身份资料 | 姓名 / 工号 / 邮箱 / 部门 | 登录与权限控制 |
| 工作内容 | OKR / KR / 1on1 笔记 / 议事室提案 / Action Items | 协作与考核记录 |
| 反馈意见 | 360 评价 / 议事投票 / 复盘 | 团队治理 |
| 知识沉淀 | Memory / 文档 / 协同表格 | 团队学习与传承 |

### 1.2 系统自动记录的

| 类别 | 例子 | 用途 |
|---|---|---|
| 访问日志 | 登录时间 / IP / 操作类型 | 审计与合规 (审计链, 见 CHARTER §2.5) |
| 设备信息 | 浏览器 UA / 屏幕分辨率 | 异常登录检测 |
| 行为埋点 | 模块使用频率 / 功能点击 | 产品改进 (默认聚合, 不个体追踪) |

### 1.3 我们 **不** 收集

- ✗ 你的私人通讯 (微信/电话/家庭住址)
- ✗ 你的浏览器历史 / 屏幕录制
- ✗ 你的生物特征 (指纹 / 面部, 除非你显式开启 MFA 设备)
- ✗ 政治倾向 / 宗教 / 工会身份等敏感数据 (PIPL §28)

### 2.1 透明度矩阵

| 数据 | 你本人 | 直属主管 | 同事 | HR | Admin |
|---|:-:|:-:|:-:|:-:|:-:|
| 个人 OKR / KR | ✅ 全 | ✅ 全 | ⚠️ 仅你公开的 | ✅ 全 | ✅ 全 |

| 1on1 主管私密笔记 | ❌ | ✅ | ❌ | ❌ | ❌ |
| 1on1 干劲评分 (moodScore) | ❌ | ✅ | ❌ | ❌ | ❌ |
| 360 评价提交 | ✅ 自己 | 汇总 | ❌ | 汇总 | 汇总 |
| 议事室决议 | ✅ | ✅ | ✅ 同议事室成员 | ❌ | ✅ |

| 审计日志 | ❌ | ❌ | ❌ | ⚠️ 仅与己相关 | ✅ |

### 2.2 透明度承诺 (CHARTER §13)

- **同事看不到你的邮箱 / IP / 锁定状态** — 已通过 EVO-7 PII 剥离框架强制实现
- **主管私密笔记永不暴露给员工本人** — 由数据库 column-level guard 强制
- **所有数据访问记录可审计** — 你可申请查看"谁在何时访问过我的数据"

## 3. 我们如何保护

### 3.1 加密

- 传输层: 强制 TLS 1.3 (生产环境)
- 静态层: 数据库 PostgreSQL 启用 pgcrypto, 敏感列 (passwordHash / mfaSecret) 永不明文存储
- 备份层: pg_dump 加密 + S3 SSE-KMS

### 3.2 访问控制

- 双因素认证 (TOTP / Yubikey 可选)
- 角色权限 (RBAC) + 数据权限 (DAC) 双层
- 审计链 SHA256 hash chain, 任何 admin 操作可追溯

### 3.3 保留期

| 数据类型 | 保留期 | 离职后 |
|---|---|---|
| 工作产出 (OKR/议事/Memory) | 永久 (公司资产) | 转移到接班人, 你的署名保留 |
| 个人偏好 / Persona | 离职即归档 | 30 天后删除, 不再训练 |
| 审计日志 | ≥ 6 个月 (等保二级要求) | 不删除 (合规需要) |
| 1on1 主管私密笔记 | 主管在职期间 | 主管离职即不可访问 |

## 4. 你的权利 (PIPL §44-47, GDPR Articles 15-22)

### 4.1 访问权 · Right to Access

- 在 `/settings/privacy` 一键导出你的全部数据 (JSON 格式)
- API: `GET /api/me/export`

### 4.2 更正权 · Right to Rectification

- 个人资料随时编辑
- 工作内容 (议事/OKR) 编辑会留版本记录, 不会"消失"

### 4.3 删除权 · Right to Erasure

- 离职流程触发自动 anonymize: 你的姓名 → "前员工 #xxxxx"
- API: `POST /api/admin/users/[id]/anonymize` (仅 admin 触发)
- 注意: 工作产出本身保留 (公司资产), 但与你的身份脱钩

### 4.4 携带权 · Right to Data Portability

- 导出 JSON 后可导入到下家系统 (开放格式, schema 公开)

### 4.5 反对权 · Right to Object

- 拒绝 Persona 训练: `/settings/persona` 关闭 "用我的工作训练 Persona"
- 拒绝 9-box 评估: 与你的主管 / HR 协商 (这通常是绩效考核合同的一部分)

### 4.6 撤回同意

- 入职时勾选的"同意隐私政策" checkbox 可在 `/settings/privacy` 撤回
- 撤回后系统会以最小数据集模式运行 (仍可登录, 但不参与 9-box / Persona)

## 5. 跨境传输

如果你的雇主将 Tandem 部署在境外 (如 AWS 美国区, Azure 香港区), 适用 PIPL §38:

- 雇主必须签订 SCCs (标准合同条款) 与 Tandem 提供方
- 必须通过国家网信办的安全评估
- 你的同意明确包含此跨境传输

如部署在境内, 不适用本节.

## 7. 政策更新

- 重大变更 (扩大数据收集 / 改变共享范围 / 跨境传输) 提前 30 天通知
- 你登录时会看到 banner "隐私政策已更新", 必须重新勾选同意才能继续使用
- 历史版本归档在 `/docs/privacy/history`
