## 一、新的战略语境

### Tandem 不是

- ❌ 从 0 起步、靠融资续命、追求 PMF 的早期 SaaS 创业项目
- ❌ 需要在 12 个月内找到第一批付费客户的产品
- ❌ 需要跟飞书 / 钉钉在公开市场拼网络效应的玩家

### Tandem 是

- ✅ **一家完整产研销企业的内部协作 AI 平台**, 类似字节早期的飞书、Tiny Speck 的 Slack、Notion 创始团队的 Notion
- ✅ **由真实业务驱动、用真实工作数据喂养、为真实同事服务**的自研工具
- ✅ **商业化是远期可选项**, 不是当下必须解决的题

### 路径类比

| 公司 | 内部自用阶段 | 对外开放 | 我们的位置 |
|---|---|---|---|
| Slack | 2009-2013 (4 年) | 2013 | 我们在第 1 年 |
| Notion | 2013-2015 (2 年闭门重写) | 2015 | 我们在第 1 年 |
| Linear | 2019 内部 → 2020 邀请制 → 2022 公开 | 2022 | 我们在第 0 年 |
| 飞书 (字节) | 2016-2020 (4 年内部) | 2020 | 我们在第 1 年 |
| GitLab | 2011 自用 → 2014 开源 | 2014 | 我们在第 1 年 |

**这是个被验证过的优秀路径**, 不是"将就"或"妥协"。

## 二、之前的判断哪些被推翻

直接对账 REFLECTION-2026-05.md 里的"红色遗憾"和"行业评分":

### 推翻的"风险"（在自用语境下不成立）

| 之前的判断 | 真相 |
|---|---|
| 🔴 "没用户没数据" 是首要死亡风险 | **失效**。Owner 公司全员就是真实用户群, 30-100+ 内部员工用 1 天的反馈比 1000 个 demo 用户值钱 |
| 🔴 "客单价 / 毛利 / ARPU 不清楚" | **失效**。不卖。AI 调用成本 = 公司内部成本中心, Owner 决定承担多少 |
| 🟡 "没多租户, tenantId 硬编码" | **失效**。单租户就够。多租户等商业化阶段再做 |
| 🟡 "企业 SSO (OIDC/SAML) 政企标配缺失" | **失效**。自用阶段用内部账号体系够 |
| 🟡 "订阅 / 计费 / 试用转化" | **失效**。不卖, 这些都不需要 |
| 🟡 "国际化 i18n" | **暂时失效**。等对外开放再做 |
| 🟢 "PWABuilder 上架 App Store" | **暂时失效**。自用 PWA 加桌面 / 主屏即可, 不需要上架 |

### 仍然成立的（在自用语境下更重要）

| 之前的判断 | 在自用语境下的修正 |
|---|---|
| 🔴 "没 CI" | **仍重要**。自用语境下不是 bus factor, 是"同事用着突然 crash 你下不了班" |
| 🔴 "AI 调用是黑盒" (LlmUsageLog) | **仍重要**, 优先级稍降。**新意义**: 公司内部成本中心要可见, 不能让 LLM 账单失控 |
| 🔴 "没用户行为数据" | **变得更重要**。自用语境下你身边就坐着 30 个真实用户, 不埋点 = 浪费金矿 |
| 🔴 "Persona 质量没评测" | **变得更重要**。同事会问"我的分身真在进化吗", 答不出 = 失去信任 |
| 🟡 "agent-runtime 没解耦" | 仍重要, 但不紧急 |
| 🟡 "lib/tools/ MCP 化" | 仍重要, 但不紧急 |

### 新出现的（自用语境下才浮现）

| 新议题 | 优先级 |
|---|---|
| 🔴 **部署稳定性**: 公司同事用, crash = 影响他们工作 | 高 |
| 🔴 **数据安全 / 备份**: 公司机密 / 议事记录 / OKR 数据丢了不可恢复 | 高 |
| 🔴 **多用户并发**: 不再是单人 dev 模式, 要支持 30-100 人同时用 | 中-高 |
| 🔴 **权限边界**: 员工 / 经理 / 老板 看到的东西要不同 | 中 |
| 🔴 **内部支持 SOP**: 同事遇到 bug 找谁? 等多久? | 中 |
| 🟡 **业务价值证明**: 用了 Tandem 后, 公司协作效率 / 决策质量 / 知识沉淀真的提升了吗? | 高 (长期) |
| 🟡 **同事采用率**: 强推还是自然采用? 不用的人为什么不用? | 高 (长期) |
| 🟡 **跟现有工具共存**: 同事原来用飞书 / 钉钉 / Notion, 怎么共存 / 替换 | 中 |

## 三、行业先进性的重新评分

之前给了 6.2/10。**在 SaaS 创业语境下这是诚实评分, 但在企业自用语境下不适用**。

新的评估应该是:

| 维度 | 自用语境下的分数 | 说明 |
|---|---|---|
| 产品哲学 | 9 | 不变, Memory + Persona + 议事 + 拿捏/搭子 创新真 |
| 核心架构 | 8 | 不变, Memory 4 层升降级 + Persona Evolution 业内独家 |
| **覆盖自用场景的完备度** | 7 | 154 路由, OKR / 议事 / 1on1 / Persona / Memory / IM / 文档 都齐了 |
| **部署可运行性** | 6 | dev 模式跑得动, 但生产部署 (HTTPS / 备份 / 多用户并发) 没演练 |
| **企业内推广就绪度** | 4 | 缺: 同事入门引导 / 权限模型 / 业务价值看板 / 故障 SOP |
| 工程基建 | 6 | 不变, 缺 CI / observability / Eval |
| **加权总分** | **6.8** | 比 SaaS 语境的 6.2 高, 因为商业化扣分项失效了 |

**对照行业**: 这个分数已经超过字节飞书在 2017 年内部使用阶段的成熟度。**不焦虑, 该往前推**。

## 四、新的下季度优先级（推翻 META-REVIEW § 八）

之前列的"下周 3 件事": LlmUsageLog / CI / 列 10 个潜在客户。**第 3 件作废**——你不需要找客户, 你就坐在客户中间。

### 新的 Top 5（按"对企业自用的边际效用"排序）

#### 🔴 1. 生产部署 + HTTPS + 备份（1-2 天）

不能再让同事用 `http://172.31.12.85:3005`。必须:

- 公司内网 / Tailscale + Caddy 反向代理 + Let's Encrypt 自签 HTTPS
- 或 VPS 部署 + 域名 + HTTPS（如 `tandem.<your-company>.com`）
- PostgreSQL 每日自动备份 (cron + pg_dump + 异地存储)
- 备份恢复演练（**至少做一次**, 否则备份等于没备份）

**为什么这是 1 号**: 同事用着的工具是真东西, 突然崩了 + 数据丢了 = 信任永远失去。

#### 🔴 2. 用户行为埋点 + 数据飞轮启动（1-2 天）

自建 `UsageEvent` 表 + 埋点关键事件:

- 谁打开了哪个页面、停留多久
- 谁触发了 OKR / 议事 / 1on1 / Persona 训练
- 谁创建了 Memory、谁审批了升级
- 谁的分身被用了几次
- 哪些功能从来没人点 (准备砍)

**为什么这是 2 号**: 你身边的 30 个真实用户每天在产生最珍贵的产品决策数据。不埋点 = 浪费金矿。

#### 🔴 3. CI（半天）

`.github/workflows/test.yml` 或 GitLab CI: push 自动跑 vitest + build + smoke e2e。

**为什么仍是 3 号**: 自用语境下不是 bus factor, 是"同事用着突然 crash 你下不了班"的实际风险。

#### 🔴 4. LlmUsageLog + 公司内部成本看板（1 天）

之前 B-005。**自用语境下的新意义**: 不是为了商业化定价, 是为了让 Owner 看到 "公司这个月 AI 用了多少钱、哪个部门用得多、是否有员工在异常调用"。

#### 🔴 5. 同事入门引导（Onboarding Tour）（1-2 天）

一个真实问题: 你的同事第一次打开 Tandem, 看到 154 路由会懵。需要:

- 首次登录的 3-5 步引导（看个视频 / 跟着做 OKR / 训练第一个 Persona）
- 一份 1 页 `docs/INTERNAL-USER-GUIDE.md` 给同事
- 每个核心模块的 "空状态" 提示（没数据时引导用户怎么开始）

**为什么这是 5 号**: 你能让同事登录 ≠ 同事会用 ≠ 同事爱用。这 3 个不同的台阶, 第 1 个已过, 第 2/3 个还没动。

### Top 5 之外的（继续在 backlog 里, 不急做）

- agent-runtime adapter (B-007)
- lib/tools MCP 化 (B-002)
- Persona Eval harness (B-008)
- 移动端 shell 重做
- ADR 12 篇
- Claude Memory 抽取接入 (B-014)
- 删 src-tauri
- 路由治理

## 五、自用阶段成功的 4 条标准

什么样算"企业自用成功", 可以考虑独立商业化?

| 标准 | 量化指标 |
|---|---|
| **采用率** | 公司 ≥ 70% 同事每周打开 Tandem ≥ 3 次, 持续 ≥ 3 个月 |
| **真实工作流跑通** | 公司 ≥ 80% 的 OKR / 议事 / 1on1 在 Tandem 完成, 不是平行存在 |
| **核心创新被真用** | Persona / Memory 被 ≥ 50% 同事主动训练 / 提升, 不只是 Owner 一人 |
| **业务价值有故事** | 至少有 3 个具体可讲的故事: "之前 X 要 2 小时, 用了 Tandem 17 分钟搞定" |

**4 条全达成** → 可以开始考虑对外开放（邀请制 → 公开 SaaS）  
**4 条达成 2 条** → 继续打磨自用, 不急对外  
**4 条达成 ≤ 1 条** → 需要回到产品定位, 检查是否方向错了

**预计时间**: 6-12 个月。**不要急。**

## 六、自用阶段不该做的事（避免分心）

| 不该做 | 为什么 |
|---|---|
| 上架 App Store / Google Play | 自己人用 PWA 就够, 上架是分散资源 |
| 写营销页 / SEO / 落地页 | 没用户来源不需要营销 |
| 设计订阅价格表 | 不卖 |
| 招市场 / 销售 / BD | 不卖 |
| 申请 ISO 27001 / SOC 2 合规 | 自己公司不需要审计自己 |
| 多语言 i18n | 同事都说中文 / 英文一种就够 |
| 多租户改造 | 单租户 (= 你公司) 就是全部 |
| 跟竞品打嘴仗 | 你不在公开市场 |
| 上 Product Hunt / 36kr 等媒体 | 自用阶段不需要曝光 |

**资源全部投入企业内推广 + 自用稳定性 + 真实数据收集**。

## 七、跟 Owner 公司其他同事的协作

既然 Owner 有"完整产研销"团队, 那么:

### 可以借力的角色

- **产品同事**: 帮做 Onboarding Tour 设计 / 用户访谈 / 业务价值故事收集
- **研发同事**: 帮做 CI 配置 / 部署运维 / 备份 SOP / 数据库优化
- **销售同事**: 暂时不需要参与 Tandem, 让他们做本职。但他们的"客户管理流程"可以作为 Tandem 的真实使用场景验证
- **HR / 行政**: 帮做内部推广 / 培训 / 收集反馈
- **运营 / 财务**: 给 LlmUsageLog 报表加财务视角

### 不该做的

- 不要把 Tandem 当 "Owner 一个人的项目" 关起门做
- 同事不是 demo 用户, 他们是真实业务上线的合作伙伴, 反馈要重视
- 不要因为"自己写的"就过度迁就, 该砍的功能砍掉 (后面会用 UsageEvent 数据决定)

## 八、给"下个 AI 协作者"的话（包括下个我）

下次 session 开始时, **先读本文档再判断**。最重要的 3 件事:

1. **不要把 Tandem 当 SaaS 创业评估**, 它是企业自用工具
2. **不要再讨论商业化** (定价 / 客单价 / GTM / 获客), 除非 Owner 主动提
3. **优先级永远是**: 同事用得稳 > 同事用得多 > 同事用得有价值 > 其它

如果 Owner 在某个会话问"商业化要怎么做", 那时再讨论, 不要主动建议。

## 九、给 Owner 自己（一句话总结）

> 你不在赛跑, 你在练功。
> 同事们的真实工作流就是道场, 6-12 个月的稳定使用 + 数据反馈 = 比任何 PMF 验证都靠谱的基础。
> 商业化是远期可选项, 不是当下必须解。
> **该做的 5 件事都不大, 但要做完: 生产部署 / 用户埋点 / CI / 成本看板 / 入门引导**。
