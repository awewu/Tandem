## 摘要 (TL;DR)

**v3 战略一刀切**: 牛马搭子 V1 **不自建任何协作底座**, 全面寄生于钉钉 / 企业微信 / 飞书.

```
✅ 自建 (差异化, 100% 工时投入):
   • OKR/TTI 双轨
   • Decision Card (决议卡)
   • 议事室 5 步骨架
   • 拿捏老板 (Persona Engine)
   • TAF (Tandem Agent Framework)
   • 四层知识架构 + Steward
   • Inbox 聚合层

❌ 不自建 (寄生 / 对接):
   • IM 消息 / 群聊
   • 邮件 (SMTP/IMAP)
   • 日历
   • 云盘 / 文档
   • 音视频会议
   • 组织架构 (镜像同步, 不编辑)
   • OA 行政 (审批 / 报销 / 印章)
   • HR / 考勤 / 外勤
   • 客户管理 / 营销
```

**理由**: V1 6-9 月预算只能投入差异化, 不能造已被三巨头打到 90 分的轮子.

## 第四章: Inbox 聚合层 (核心创新)

### 4.1 设计思路

员工最大痛点: **多个 IM 平台疲于奔命**.

牛马搭子 Inbox 不替代 IM, 而是**聚合"决策类信息"**:

```
┌─────────────────────────────────────────┐
│  牛马搭子 Inbox (聚合层)                 │
├─────────────────────────────────────────┤
│  📨 钉钉 @我的 (待回应)         3 条     │
│  📨 企微 @我的 (待回应)         5 条     │
│  📨 飞书 @我的 (待回应)         2 条     │
│  📧 邮件 (待回应)               7 封     │
│                                         │
│  📋 议事室待开会                2 场     │
│  📊 Decision Card 待签         4 张     │
│  ✅ Action Items 我负责        9 项     │
│  📈 KR 待 Check-in              3 个     │
│                                         │
│  AI 已为你聚合, 按"必须回应优先级"排序   │
└─────────────────────────────────────────┘
```

### 4.2 关键能力

- **一站式回应**: 不离开牛马搭子可回钉钉/企微/邮件
- **AI 优先级**: 哪些是真紧急, 哪些可推迟
- **统一异步处理**: 配合心流保护, 每天 2-3 次集中处理
- **决议入口**: 任何 IM 消息可一键转 Decision Card

### 4.3 实现路径

```
V1: 单向聚合 (拉)
  • 拉钉钉/企微/飞书的 @我的消息
  • 拉邮件
  • 在 Inbox 展示

V2: 双向同步 (回写)
  • 在 Inbox 直接回复, 推送回原平台
  • 标记已读同步

V3: 智能聚合
  • 多消息合并语义
  • 自动生成决策建议
```

### 4.4 与 MANIFESTO 第十一条联动

第十一条说"反对消息黏性, 拥抱异步聚合". Inbox 是这条的产品落地.

   • 拿捏老板权限分级
   • 议事室参会者推荐
