# TTI 框架白皮书

## Target to Improve · 目标改善体系

> 一种面向 AI 时代中国职场的 OKR 落地方法论
>
> 版本: v1.0
> 最后更新: 2026-05
> 作者: 牛马搭子 (Tandem) 产品团队

## 摘要 (TL;DR)

**TTI (Target to Improve)** 是牛马搭子产品团队在 OKR 中国本土化实践中提出的一套**双轨绩效管理框架**.

它的核心主张是: **KPI 决定钱, TTI 决定成长**. 二者**彻底分离, 互不污染**.

```
   ⚙️ KPI (刚性管理)            🚀 TTI (改善牵引)
   ──────────────              ──────────────
   性质: 必须达成               性质: 拉伸挑战
   合格线: 100%                 健康区: 60-70%
   挂钩: 奖金/调薪/末位         挂钩: 述职/晋升/高潜
   问题: 你做对了吗?            问题: 你超越了吗?
```

TTI 的诞生, 是为了解决 OKR 在中国落地的根本悖论:

- **Google 模式** (OKR 完全不挂绩效) → 中国老板不放心, 落地难
- **Tita 模式** (OKR 直接挂奖金) → OKR 异化为"另一种 KPI", 失去拉伸价值

TTI 框架以**双轨独立**为核心, 让 KPI 给员工"安全感", TTI 给员工"想象力", **解决了"老板心安 + 员工心宽 + 卓越者有通道"的中国职场三重平衡难题**.

## 第一章: 我们为什么需要 TTI

### 1.1 OKR 在中国的两难

OKR 进入中国十多年, 头部互联网公司大多尝试过, 但落地结果两极分化:

**保留 OKR 拉伸哲学的公司**: OKR 形同虚设, 沦为季度填空作业.
**严肃执行 OKR 评分的公司**: OKR 被员工写得越来越保守, 异化为"另一种 KPI".

根本原因不是工具, 是**激励结构**:

```
┌─────────────────────────────────────────────────┐
│  Google 模式 (西方原版)                          │
│                                                 │
│  OKR ──── 不挂绩效 ──── 文化驱动                 │
│   │                       │                     │
│   ▼                       ▼                     │
│  员工敢写大目标          老板信任员工            │
│  (因为不影响奖金)        (因为有强公司文化)      │
│                                                 │
│  在中国的问题:                                   │
│  ❌ 文化未建立, 老板不放心                       │
│  ❌ 销售/客服/工厂等岗位无法适用                 │
│  ❌ 员工不知道做这个有什么用                     │
└─────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────┐
│  Tita 模式 (中国本土化)                          │
│                                                 │
│  OKR ──── 直接挂奖金 ──── 工具兜底               │
│   │                       │                     │
│   ▼                       ▼                     │
│  员工把 OKR 写得保守     老板觉得有抓手          │
│  (因为不想丢钱)          (因为可以打分)          │
│                                                 │
│  在中国的问题:                                   │
│  ❌ OKR 异化, 失去拉伸价值                       │
│  ❌ "60-70% 完成是健康"哲学崩塌                   │
│  ❌ 变成"另一种 KPI", 无差异化                   │
└─────────────────────────────────────────────────┘
```

### 1.2 为什么 KPI 不能丢

许多 OKR 倡导者主张"用 OKR 替代 KPI". 这在中国是错误的.

中国职场需要 KPI 的原因:

1. **岗位本质**: 销售、客服、工厂等岗位的工作产出本就是数字化、刚性的. KPI 是其天然语言.
2. **管理传统**: 中国老板需要"硬抓手", 完全软性激励无法落地.
3. **员工预期**: 中国员工需要"明确预期 → 明确回报"的契约感, 模糊激励不被接受.
4. **法律边界**: 末位淘汰、PIP 等需要客观数据支撑, KPI 是法律可辩护的依据.

**KPI 不应被废, 应被升级**. 升级方向是: 让 KPI 设定环节更严肃, 让 KPI 数据采集更客观, 让 KPI 不再"拍脑袋".

### 1.3 为什么 OKR 也不能丢

但只要 KPI, 公司又会陷入"内卷"陷阱:

- 员工只做 KPI 内的事, 拒绝挑战
- 创新无人发起 (因为创新失败影响 KPI)
- 跨部门协作难以推动 (与 KPI 无关)
- 卓越者无差异化通道 (KPI 完成 100% 大家一样)

OKR 的真正价值不是"打分", 而是**给追求卓越的员工一个独立通道**.

### 1.4 TTI 的破局

TTI 框架的核心创新:

> **OKR 不是用来打分的, 是用来识别人的.**

我们保留 OKR 的拉伸哲学, 但改换它的归宿:
- 不挂奖金 → 不再异化
- 挂述职 + 晋升 + 高潜 → 仍有真实激励
- 60-70% 完成是健康 → 拉伸价值保留
- 失败也获经验 → 鼓励试错

并把它**改名为 TTI (Target to Improve)**, 强调"目标是用来改善的, 不是用来完成的". 一字之差, 哲学根基立刻不同.

## 第二章: TTI 的完整定义

### 2.1 TTI 是什么

**TTI (Target to Improve)** 是员工为驱动**自身或组织改善**而设定的拉伸性目标.

特征:

| 维度 | TTI 定义 |
|---|---|
| 目的 | 改善 (improve), 而非达成 (achieve) |
| 健康完成度 | 60-70% (100% 触发"目标定低"提示) |
| 周期 | 季度 / 半年 / 年度 |
| 失败成本 | 学习 (失败也获经验值) |
| 公开度 | 全公司透明 |
| 关联 | 关联 KR / 决议 / 案例库 |
| 挂钩 | 述职 / 晋升 / 高潜 / IDP |
| 不挂钩 | 奖金 / 调薪 / 末位 |

### 2.2 TTI 与 OKR/KR 的关系

**TTI 是 OKR/KR 的中国语境化命名**, 哲学血缘一致, 但落地姿态不同.

| 概念 | OKR/KR (西方) | TTI (中国语境) |
|---|---|---|
| 字面 | Objective + Key Result | Target to Improve |
| 暗示 | "Key" 暗示必须达成 | "Improve" 明确允许不达成 |
| 中文翻译 | 关键结果 | 改善目标 / 拉伸指标 |
| 文化重心 | 跨部门对齐 | 个人成长 + 组织改善 |
| 本质 | 同一思想 | 同一思想 |

在牛马搭子产品中, **TTI 与 OKR/KR 共存**:

- 数据模型层: 仍叫 Objective + KeyResult (兼容主流标准)
- 用户体验层: 命名为 TTI (中国职场友好)
- 方法论层: TTI 哲学覆盖整个体系

### 2.3 TTI 的三个组成

```
┌──────────────────────────────────────────────────┐
│             TTI 体系完整结构                      │
├──────────────────────────────────────────────────┤
│                                                  │
│  📍 改善方向 (Improvement Direction)              │
│     一句话定性表述, 类似 Objective                │
│     例: "提升新用户冷启动体验"                    │
│                                                  │
│  📊 改善指标 (Improvement Metrics)                │
│     可衡量, 类似 Key Result                       │
│     例: "首次激活率从 35% → 50%"                  │
│                                                  │
│  🎯 改善行动 (Improvement Actions)                │
│     具体推动, 类似 Initiative                     │
│     例: "重构 onboarding 流程 v3"                 │
└──────────────────────────────────────────────────┘
```

### 3.2 双轨员工分流模型

TTI 框架自然把员工分成两类, 互不强制:

```
🛋️  满足现状型 (Maintainers)
   • 主要走 KPI 通道
   • 拿奖金, 安稳过日子
   • 不写 TTI 也没关系
   • 公司允许这类员工存在 (这是尊重)

🚀  追求成长型 (Aspirants)
   • 同时走 KPI + TTI
   • KPI 兜底安全
   • TTI 拼晋升和高潜
   • 自驱写好 TTI (因为他们想晋升)
```

**这是对员工最大的尊重** —— 不强迫所有人都"卷", 让追求不同的员工各取所需.

### 3.3 一句话员工契约

向员工沟通时, **不需要培训, 不需要改文化**, 只用一句话:

```
KPI: 你必须做到的 (做不到, 工作不保)
TTI: 你想超越自己 (做到了, 走得更远)

KPI 给你"安全感"
TTI 给你"想象力"
```

## 第四章: 9 宫格人才矩阵

### 4.1 KPI × TTI 人才矩阵

借助经典 GE/McKinsey 9-Box Grid, 把 KPI 和 TTI 双轨结果自然映射到人才管理:

```
                          KPI (实绩) →
              不达标 (<100%)   达标 (100%)    超出 (>100%)
         ┌─────────────────┬──────────────┬─────────────────┐
   突出  │ ❓ 待开发高潜    │ 💎 高潜稳健   │ ⭐ 核心接班人    │
  TTI    │  Hidden Gem      │  Rising Star  │  Successor       │
  (潜力) │  → 调岗/给机会    │ → 加速培养    │ → 高曝光项目     │
   ↑     ├─────────────────┼──────────────┼─────────────────┤
   中    │ ⚠️ 方向错位     │ 🟢 中坚骨干   │ 🚀 强力贡献者    │
         │  Misaligned      │  Core Solid   │  High Performer  │
         │  → 1:1 校准       │ → 稳定激励    │ → 提拔候选       │
         ├─────────────────┼──────────────┼─────────────────┤
   低    │ 🔴 末位          │ ✅ 稳定老将   │ 🎯 技术专家      │
         │  PIP / 改进       │  Reliable     │  Specialist      │
         │  → 警示或淘汰     │ → 维护现状    │ → 专家通道       │
         └─────────────────┴──────────────┴─────────────────┘
```

### 4.2 9 个格子的管理动作建议

| 格子 | 解读 | 推荐管理动作 |
|---|---|---|
| ⭐ 核心接班人 | 高 TTI + 超 KPI | 高曝光项目 / 跨部门轮岗 / 晋升序列加速 |
| 💎 高潜稳健 | 高 TTI + 达 KPI | IDP 培养 / 导师配对 / 战略项目机会 |
| ❓ 待开发高潜 | 高 TTI + 低 KPI | 重新 1:1 校准 KR / 是否岗位错配? 调岗试试 |
| 🚀 强力贡献者 | 中 TTI + 超 KPI | 提拔候选 / 加速职级 / 增加 TTI 拉伸 |
| 🟢 中坚骨干 | 中 TTI + 达 KPI | 稳定激励 / 维护士气 / 适度挑战机会 |
| ⚠️ 方向错位 | 中 TTI + 低 KPI | 1:1 校准方向 / 检查 KPI 设定合理性 |
| 🎯 技术专家 | 低 TTI + 超 KPI | 专家通道 / 技术影响力扩展 / 不强求晋升 |
| ✅ 稳定老将 | 低 TTI + 达 KPI | 维护现状 / 价值认可 / 不强推 TTI |
| 🔴 末位 | 低 TTI + 低 KPI | PIP 改进 / 警示淘汰 (基于 KPI 不达成) |

**核心原则**:

- **末位淘汰仅基于 X 轴 (KPI)**, 不基于 Y 轴 (TTI)
- TTI 不达成不会触发任何惩罚, 只是不进入高潜池
- 9 个格子每个员工每季度自动归位, AI 推荐管理动作

### 4.3 9 宫格在产品中的 4 种用法

```
1️⃣ 个人视图: 员工自己看到自己在哪格 (透明)
   → "我是 🟢 中坚骨干, 想冲 ⭐ 核心需要在 TTI 上突破"

2️⃣ 经理视图: 团队每位员工的位置 (一张图掌握团队)
   → "我团队 12 人, 3 个高潜在 💎/⭐, 1 个 🔴 需介入"

3️⃣ HR 视图: 全公司分布 (健康度)
   → "🟢🚀 应占 40-50%, 🔴 应 < 5%, 比例失衡需诊断"

4️⃣ Calibration 视图: 跨团队 9 宫格分布对比
   → 校准会自动用此图, 避免某经理给分过松/过严
```

## 第五章: TTI → 述职 → 高潜的完整通道

### 5.1 季度述职 (Quarterly Talent Review)

**这是 TTI 真正的归宿**.

```
┌──────────────────────────────────────────────────┐
│         季度述职会 (基于 TTI + KPI)               │
├──────────────────────────────────────────────────┤
│                                                  │
│  时长: 30 分钟 / 人                              │
│  参会: 员工 + 直属上司 + skip-level + HRBP       │
│  形式: 员工本人主讲, AI 提供数据底板              │
│                                                  │
│  议程:                                           │
│  1️⃣ 5 min: KPI 完成情况 (硬数据, AI 一键展示)    │
│  2️⃣ 15 min: TTI 推动情况 (员工讲故事)            │
│      • 关键决议 (Decision Cards)                 │
│      • 突破时刻 (60-70% 完成的拉伸经验)          │
│      • 失败教训 (鼓励诚实)                        │
│      • 跨部门贡献                                │
│  3️⃣ 5 min: 自我成长评估 (拿捏老板成长曲线)       │
│  4️⃣ 5 min: 下季度 TTI 设定 + 资源诉求            │
│                                                  │
│  AI 角色:                                        │
│  • 会前: 自动生成述职材料包                      │
│  • 会中: 数据 fact-check (Decision Card 溯源)    │
│  • 会后: 高潜评分写入 HR 系统                    │
└──────────────────────────────────────────────────┘
```

**痛点解决**:
- 员工不再加班写 PPT (AI 自动生成)
- 内容数据驱动, 不是文字玄学
- 上司 5 分钟读完, 签字
- HR 拿到的是结构化数据, 不是 PDF 长文

```
│   • 案例库贡献仅 2 条 (低于团队均值)            │
│                                                 │
│ ▶ 下季度 TTI 计划                              │
│   • TTI-007 ...                                │
│                                                 │
│ ▶ 9 宫格自我定位: 💎 高潜稳健                   │
│                                                 │
│ [审阅 / 修改 / 提交至 HR 系统]                 │
└─────────────────────────────────────────────────┘
```

### 5.3 高潜识别 (HiPo Pool)

高潜评分基于 TTI 数据 + 9 宫格位置:

```
高潜评分体系

┌─────────────────────────────────────────────┐
│  维度          指标                          │
├─────────────────────────────────────────────┤
│  9 宫格位置     ⭐/💎/❓ 入选, 其他不入       │
│  挑战度        TTI 设定的 ambition score    │
│  完成度        TTI 实际完成 (健康区 0.6-0.8) │
│  影响范围      推动了几个跨团队 KR           │
│  决议质量      Decision Card 采纳率         │
│  失败学习      失败 TTI 的复盘深度           │
│  自创创新      🅳 自创方案被采纳的次数       │
│  知识沉淀      贡献到 Memory 层的次数       │
│  分身成长      拿捏老板成长阶段             │
└─────────────────────────────────────────────┘

→ 综合分 → 高潜池 (Top 10%-20%)
→ 自动触发 IDP 培养计划
→ 内部岗位机会优先匹配
```

### 5.4 Calibration 校准会的 AI 升级

HR 最痛的环节: 经理们坐在一起争"我组小王是 A 还是 A+".

牛马搭子的输出:
- **客观数据墙**: 每个候选人 KPI/TTI/决议三维数据并列
- **风险检测**: AI 提示"小王和小李 KPI 接近但小王 TTI 推动了 3 倍"
- **历史对比**: 和上季度同等级员工的数据对照
- **盲打分**: 屏蔽姓名/性别/年龄做 9 宫格盲分类

## 第六章: TTI 实施指南

### 6.1 实施前提 (Prerequisites)

实施 TTI 框架需要满足以下条件:

| 条件 | 说明 |
|---|---|
| 公司有清晰战略 | TTI 上挂战略, 没战略 = 没 TTI |
| 有可数据化 KPI | KPI 设定可量化、可自动采集 |
| 一定 OKR 文化基础 | 至少经理层听过 OKR (无要求精通) |
| 接受"60-70% 健康"哲学 | 不能要求 TTI 100% 完成 |
| 高层愿意公开 OKR | 至少 C-level 公开 |

### 6.2 实施路径 (4 阶段)

```
阶段 1: 战略部署 Workshop (1-2 天)
  ↓
  • CEO 注入年度战略 → 公司级 TTI
  • 部门 → 团队 → 个人 TTI 拆解
  • KPI 设定 (与岗位说明书绑定)
  • 三审视过滤 (历史/客观/可控)

阶段 2: 季度执行 (3 个月)
  ↓
  • 周报 + KR Check-in
  • 议事室决议 → 关联 TTI
  • Decision Card 沉淀

阶段 3: 季末述职 (1 周)
  ↓
  • AI 生成述职材料包
  • 30 分钟会议, 评估 9 宫格位置
  • TTI 进入述职归档

阶段 4: 跨季校准 (1 周)
  ↓
  • Calibration 会, 盲打分校准
  • 高潜池更新
  • IDP 调整
  • 下季 TTI 设定
```

### 6.3 常见误区

| 误区 | 正解 |
|---|---|
| "TTI 100% 完成是好事" | ❌ 反思目标定低; 60-70% 是健康 |
| "TTI 应该挂奖金, 不然没动力" | ❌ 双轨独立, TTI 挂晋升即足够 |
| "TTI 应该全员强制写" | ❌ 自愿写, 想晋升的写就够了 |
| "KPI 和 TTI 应该一起算绩效分" | ❌ 永不相加, 各管一边 |
| "9 宫格用来淘汰末位" | ❌ 末位仅基于 X 轴 (KPI), 不看 Y 轴 |

## 第七章: 与同类方法论的对比

### 7.1 对比表

| 维度 | OKR (Google) | OKR (Tita) | TTI (牛马搭子) |
|---|---|---|---|
| 起源 | Intel/Google | 中国本土 | 牛马搭子产品团队 |
| 是否挂钱 | 完全不挂 | 直接挂 | 完全不挂 |
| 是否挂晋升 | 间接 | 综合考虑 | 直接挂 |
| 完成度健康 | 0.6-0.7 | 越高越好 | 60-70% |
| 失败容忍 | 高 | 低 (扣分) | 高 (失败也获经验) |

| 中国适配 | 弱 | 强 (但异化) | 强 (不异化) |

| 适用人群 | 注重成长的全员 | 全员强制 | 自愿: 想成长的员工 |

### 7.2 TTI 的独特定位

TTI 既不是 Google 的纯哲学派, 也不是 Tita 的纯产品派, 是**面向 AI 时代中国职场土壤的第三道路**:

- **保留**: Google 的 60-70% 健康哲学、全公司透明、Learn from misses
- **接纳**: 中国职场对 KPI 刚性的天然需求
- **创新**: TTI 命名让员工天然理解; 9 宫格直映射人才管理; AI 决策支持深度集成

## 结语: TTI 是反卷的工具, 也是赋权的工具

TTI 框架的最深价值, 不在于方法论本身的精巧, 而在于它**重新分配了员工的自由度**:

- 满足现状者, 可以不写 TTI, 走 KPI 通道, 体面拿钱
- 追求成长者, 可以写 TTI, 走述职通道, 突破上升

它**反卷**: 因为不强迫所有人都拉伸目标
它**赋权**: 因为给追求成长者一条独立通道
它**保护**: 因为 KPI 兜底让员工不必拼命卷

> **「TTI 给敢做梦的人一个公平的舞台」**

这是我们对中国职场的承诺.

## 附录 B: TTI 9 宫格归格规则

```
KPI 轴 (X):
- 不达标: 实际 / 目标 < 100%
- 达标:   实际 / 目标 = 100% ± 5%
- 超出:   实际 / 目标 > 105%

TTI 轴 (Y):
- 低:     完成度 < 40% 且无突破性贡献
- 中:     40-60% 完成 OR 60-80% 但无突破
- 突出:   60-80% 完成 + 至少 1 个突破性 Decision Card
          OR 80-100% 高质量完成

突破性 Decision Card 定义:
  • 自创方案 🅳 被采纳
  • 推动跨部门 KR
  • 沉淀成 Memory (经签批)
  • 案例库 Best Practice 入选
```
