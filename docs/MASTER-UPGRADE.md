## 3. 升级路线 · 3 里程碑 × 5 体系

**不是功能清单堆叠, 是体系能力跃迁。所有功能保留在蓝图, 按依赖分阶段。**

### M1 连通 (4-5 周) — 5 体系打通最小闭环

| 体系 | 升级动作 | 对标来源 | 复用现有 | 新增 |
|------|----------|----------|----------|------|
| **AI 赋能** | 全员 Ask Tandem 入口 + Cmd+K 面板 + 问答历史 | Claude/搭子 | CompanyBrain API | 前端 UI + `ai_conversations` 表 |
| **战略执行** | 级联 inline 编辑 + 季度复盘流 + 1on1 会前议程 | Tita | 级联视图/DC/日报 | 可编辑 state + 聚合逻辑 |
| **知识治理** | `/workspace` 统一空间 + 全局搜索回归 | Notion | 文档/表格/Memory | 统一壳 + `/api/search` |
| **沟通协同** | 邮箱 IMAP 收件 + 审批→IM 闭环 | Gmail/WeCom | SMTP/IM 流 | node-imap + 审批卡片 |
| **组织治理** | 1on1 与 OKR 联动 + 全链路审计可见 | Tita | 1on1/审计 hash | 联动逻辑 |

### M2 深化 (3-4 周) — AI 成为默认交互层

| 体系 | 升级动作 |
|------|----------|
| **AI 赋能** | 上下文感知推荐 + AI 生成日报/复盘/邮件 (Persona 口吻) |
| **战略执行** | 信心指数趋势 + 多周期对比 + 绩效面谈模板 |
| **知识治理** | 块编辑器 (Tiptap Block) + 看板/日历视图 |
| **沟通协同** | IM 响应式 + 语音消息 + 邮箱线程视图 |
| **组织治理** | 成长仪表盘 + XP 系统 + 成长预测 |

### M3 涌现 (3-4 周+) — 体系联动产生涌现

| 联动 | 涌现效应 |
|------|----------|
| 战略 × AI | OKR 自动发现偏差 → 自动 spawn 议事室 → 自动生成复盘 |
| 知识 × AI | 文档自动提取 Memory 候选 → 自动 propose 升级 |
| 沟通 × AI | IM 消息自动识别议题 → 一键 spawn DC |
| AI × 知识 | 问答自动沉淀 Memory → 反哺 Baseline |
| 组织 × AI | 1on1 前自动议程 → 会后自动成长建议 |

## 5. 5 个关键架构决策 (一旦定不回头)

| 决策 | 选项 | 定论 |
|------|------|------|
| AI 入口形态 | 悬浮球 / 搜索栏+Cmd+K / 模块内嵌 | **M1 做搜索栏+Cmd+K, M2 叠加模块内嵌, 不做悬浮球** |
| 知识空间统一 | 外套壳 / 深度整合 / 仅搜索 | **M1 外套壳快速统一, M2 渐进深度整合** |
| 移动端 | 响应式 / PWA / 原生 App | **M1 响应式, M2 评估 PWA, 不做原生** |
| 块编辑器 | 升级 Tiptap / 换 Editor.js / 保持 | **M1 保持现有, M2 升级 Tiptap, 不换引擎** |
| 成长可视化 | XP 进度条 / 技能树 / 3D Avatar | **M1 XP, M2 技能树(无排行榜), 不做 Avatar** |

## 6. 工作量与价值

### 6.1 工期

| 阶段 | 串行 | 2 人并行 |
|------|------|----------|
| M1 连通 | 4-5 周 | 2.5-3 周 |
| M2 深化 | 3-4 周 | 2-2.5 周 |
| M3 涌现 | 3-4 周+ | 2-2.5 周 |
| **合计** | 10-13 周 | **6.5-8 周** |

### 6.2 价值 (业务量化)

| 受众 | 价值点 | 量化 |
|------|--------|------|
| 老板 | 决议效率 | 会议 60min → 17min, 省 70% |
| 老板 | 执行诚信 | 虚报率 30% → <5% |
| 员工 | 反卷 | 日报省 15-20min/天 |
| 员工 | 成长可见 | 反馈周期 季度 → 每天 |
| Steward | 治理透明 | 采纳率/成本/延迟全可见 |

### 6.3 产品定位跃迁

> 从"功能齐全的协作工具" → "**AI 驱动的企业决议操作系统 + 员工成长伴侣**"

## 7. 执行纪律 (防发散 5 条)

1. **每阶段 ≤4 并行任务**, 超出放下阶段
2. **复用现有数据模型**, 新功能落 KvStore, 不加 Prisma schema (禁 drizzle push)
3. **UI 走 semantic token**, 不引入 raw Tailwind / 新设计风格
4. **先连线再新增**: 先把现有分散能力串成闭环
5. **宪章 v2.0 三红线**: 双轨分离(成长/状态不挂钱晋升) + 公平底线(不羞辱/不抹黑) + 中央基线管控(个人 AI 出产必经 4 道闸)。其余大胆吸收竞品灵魂。
