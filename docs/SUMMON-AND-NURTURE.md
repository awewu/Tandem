## 一、 板块概念对齐 (Consensus Alignment)

为符合产品宪章的语义语境，我们将双板块的功能边界对齐为 **「拿捏 = 个人养成」** 与 **「搭子 = 工具召唤」**：

```
                              Tandem AI
                                  │
         ┌────────────────────────┴────────────────────────┐
         ▼                                                 ▼
     【 拿捏 · 个人成长 】                             【 搭子 · 召唤搭子 】
     (Persona Nurturing)                                (Agent summoning)
         │                                                 │
  养我自己的“龙虾与分身”                              召唤公司配置的标准专家
  · 1-5 阶段自我进化                                  · 智能体超市 (Agent Market)
  · 提取我的日报/周报作为养料                           · 战时作战台 (模板化协作)
  · 绿区安全代行 (24h 否决 §9)                         · 协同产出一键沉淀为决议
```

## 二、 6 大开放问题战略决议 (Solidified Decisions)

在首期设计研讨中，我们对以下 6 个底层安全、隐私与协同边界达成了终极决议：

### 1. 命名与心智对齐 (Naming Alignment)

- **决议**：采用 **方案 A**。保持当前导航命名不变（`me` 对应 `拿捏 · 个人成长`；`ai` 对应 `搭子 · 召唤搭子`）。
- **心智故事**：
  - **拿捏**：解决的是“自我掌控与职业成长”——我如何训练我自己的分身、积累技能，直到在休假时分身能优雅帮我代行，**拿捏我的职业未来**。
  - **搭子**：解决的是“协同干活与生产力”——我今天需要一个专业的法务、研发或销售顾问，我随时在超市里 **召唤一个 AI 英雄当我的工作搭子**。

### 2. 智能体治理与防止野生 Agent (Agent Governance)

- **决议**：**管理员（Admin）集中配置为主数据 + 允许个人 Fork 修改**。
- **机制**：
  - 公司通过 `/admin/launchpad` 和 `TAF Skills` 配置统一的销售、研发、法务标准智能体，确保基线（Baseline）对齐。
  - 允许员工对标准 Agent 进行个人 Fork（如：销售 Agent 加上员工自己负责的垂直渠道偏好），但此 Fork 版本默认仅自己可见，不得私自向全员发布，防止产生“野生 Agent 逻辑冲突与漂移”。

### 3. 分身被同事召唤与绿区代行 (Persona Delegation)

- **决议**：**能被同事召唤或代行，但安全边界严格按照宪章 §9 铁律**。
- **机制**：
  - 当我休假或离线时，同事可以在 IM 或邮件中 `@我的分身` 寻求代行。
  - **绿区（可代行）**：解答关于我负责项目的常规状态、SOP 规定内的常规确认。
  - **黄区（需人工）**：涉及轻微排期调整、需要判断的选择。分身会代行起草草稿，推送至我的 Inbox 并在我上线时一键确认。
  - **红区（严禁代行）**：客户谈判、招聘面试、绩效面谈、法务合规审计。分身直接自动拒绝代行，并提示“由于红线规定，该事务已升级留待本人处理”。
  - **终极防线**：任何代行产物，员工本人在上线后均拥有 **24 小时硬否决（Denial Window）窗口**，否决后系统自动通知关联方撤销。

### 4. 协作产物的默认隐私保护 (Privacy Guard)

- **决议**：**默认 100% 隐私，只有员工主动授权才进入公司公域数据**。
- **机制**：
  - 员工在「搭子」或「拿捏」板块中与 AI 的任何讨论，其对话流、临时文档默认只有员工本人可见，Steward、Admin 和主管在后台均无权检索或查看（守住宪章 §13.2 员工尊严）。
  - 只有当员工在工作台中点击 **【一键转为决议卡】** 或 **【沉淀到 Material 知识库】** 时，该协作成果才进入公司多租户（Tenant）公域数据中，并留下审计。

### 5. 分身训练数据的自愿选择权 (Opt-In Training)

- **决议**：**100% 自愿 Opt-In，允许一键自助关停或抹除**。
- **机制**：
  - 分身在 `/persona` 训练时所汲取的数据源（日报、周报、会议记录、OKR），员工可在 `/settings/privacy`（宪章 §13 数据自助页）中自主选择打开或关闭。
  - 支持“一键擦除我的分身记忆链”，满足数据可携权与遗忘权。

### 6. 并排测试与双答案竞技场 (Comparison Arena)

- **决议**：**V1 阶段单独跑通各自闭环，双答案并排测试放入 V2 迭代**。
- **机制**：
  - V1 阶段专心跑通「标准 Agent 工作台」和「个人分身进化与训练台」。
  - V2 阶段再引入“PK 模式”（如：对一个客户投诉，让「公司标准售后 Agent」和「你养的资深分身」并排给出方案对比）。

### 7. 拥抱市面个人 AI · Skill Gateway 接入 (新增 2026-05-27)

> 来源: `docs/OKR-DRIVEN-ARCHITECTURE.md` § 三 第 3 条 + `docs/MANIFESTO.md` 第十九条.

- **决议**: **拿捏 + 搭子两个板块均开放接入市面所有个人智能体**, 不重新发明个人 AI. Tandem 做组织级网关, 不做个人 AI 竞品.
- **接入清单 (示例, 非穷举)**:
  - **代码类**: Claude Code / Cursor / Codex / Hermes Agent / OpenClaw
  - **文档类**: Notion AI / 飞书 AI / 钉钉 AI / WPS AI
  - **通用类**: ChatGPT / Claude / DeepSeek / Kimi / Manus
- **接入路径** (V2 启动 B-016 + B-017):
  1. **Capture 层 (B-016)**: 个人 AI 产出 (代码/文档/决议草稿) 通过 IDE 插件 / 邮件 webhook / 文档 metadata 回流到 Tandem Material 个人级
  2. **Skill Gateway 4 道闸 (B-017)**: 个人 AI 调企业数据 / 工具时穿过 ① Baseline-Guard ② OKR Drift ③ Data Scope ④ Action Scope
- **机制**:
  - 拿捏板块: Persona 训练数据可包含 "个人 AI 跟我对话历史" (员工 opt-in)
  - 搭子板块: Agent Marketplace 列出"市面 AI 接入选项", 跟"公司标准 Agent"并列, **但所有调用进 Skill Gateway 网关**
- **不重新发明**:
  - ❌ 不做"Tandem 自带 Coder Agent" 跟 Cursor/Claude Code 竞争
  - ❌ 不做"Tandem 自带 Writer Agent" 跟 Notion AI 竞争
  - ✅ 做"Tandem 是个人 AI 的组织级网关 + 反哺 Capture 层"

## 三、 V1 阶段双板块待交付功能 (Feature Backlog)

### 1. 【搭子】 召唤板块功能清单

- **A1. 智能体超市 (Marketplace UI)**：分类展示公司预配的标准 Experts。
- **A2. “作战室”模板工作台 (Work Templates)**：
  - 选择 Agent 后，提供并排分屏：左侧为业务实体（如 PRD 文档、合同文本、代码片段、数据表），右侧为流式 AI 讨论区，直接实时对业务实体进行高亮或修改。
- **A3. 一键沉淀决议 (沉淀按钮)**：讨论出的 Action Items 一键反推到 `addCheckIn` 或者是 `Decision Card`。

### 2. 【拿捏】 养成板块功能清单

- **B1. 训练台对话界面 (Training Ground)**：员工和分身纠偏、标注“这不像我”反向校正的对话流。
- **B2. 养料来源仪表盘 (Data Sourcing)**：透明展示分身当前学了我的哪些日报、周报和决议，允许一键 opt-out。
- **B3. 五阶段进化地图与里程碑记录**：让“龙虾/Hermes”的成长可视化。
- **B4. 绿黄红区代行边界清单配置**：员工可以手动修改分身代行回复的敏感边界。
