# 数据处理协议 (DPA) · 模板

> 适用范围: 企业客户 (数据控制者) 与 Tandem 提供方 (数据处理者) 之间.
> 用法: 私有化交付前, 双方法务审阅并签署. 本文档为骨架, 具体条款由律师定稿.
> 法律框架: PIPL (中国大陆) / GDPR (欧盟) / 其他地区适用法.

---

## 1. 定义

| 术语 | 定义 |
|---|---|
| 数据控制者 | 客户企业 (决定个人信息处理目的与方式) |
| 数据处理者 | Tandem 提供方 (受托代为处理) |
| 数据主体 | 客户的员工 |
| 个人信息 | PIPL §4 定义 |
| 敏感个人信息 | PIPL §28 定义 (生物识别 / 宗教 / 医疗 等) |

---

## 2. 处理范围

### 2.1 数据类型 (双方核对勾选)

- [ ] 员工身份资料 (姓名/工号/邮箱/部门/职级)
- [ ] 工作产出 (OKR/KR/议事/Memory/文档)
- [ ] 1on1 / 360 反馈
- [ ] KPI 绩效数据
- [ ] Persona / AI 行为画像
- [ ] 访问审计日志
- [ ] 其他: __________

### 2.2 处理目的

仅限于:
1. 提供 Tandem 协作服务
2. 客户委托的合同义务履行
3. 法律法规要求的合规存档
4. **不得用于**: 处理者自身营销 / 二次销售给第三方 / 训练通用模型

### 2.3 处理时长

- 同合同期 + 离职后保留期 (默认 90 天, 可调)
- 合同终止后, 90 天内迁移交还客户; 之后 30 天内安全销毁

---

## 3. 处理者义务

### 3.1 安全措施 (PIPL §51, GDPR Art 32)

- [x] 静态加密 (PostgreSQL TDE + S3 SSE-KMS)
- [x] 传输加密 (TLS 1.3)
- [x] 访问控制 (RBAC + DAC)
- [x] 审计日志 (SHA256 hash chain, ≥6 个月留存)
- [x] 灾难恢复 (RTO < 4h, RPO < 24h)
- [x] 渗透测试 (年度)
- [x] 漏洞响应 SLA: 高危 24h / 中危 7d / 低危 30d

### 3.2 子处理者 (Sub-processor)

如 Tandem 使用以下子处理者, 需控制者书面同意:

| 子处理者 | 用途 | 国家/地区 | 适用合同 |
|---|---|---|---|
| AWS / Azure / 阿里云 | 基础设施 | 客户指定 | DPA 转交条款 |
| DeepSeek / Qwen | LLM 推理 | 中国 | 数据不入训练集 |
| (其他) | __________ | __________ | __________ |

**默认配置: 自托管, 不调用境外服务.**

### 3.3 数据主体权利支持

处理者承诺协助控制者在以下时限内响应数据主体请求:

| 请求类型 | 响应时限 |
|---|---|
| 访问 / 导出 | 15 工作日 |
| 更正 | 5 工作日 |
| 删除 / 匿名化 | 30 天 |
| 停止处理 | 立即 |

### 3.4 泄露通知

- 处理者发现疑似泄露后 **24 小时内** 通知控制者
- 协助控制者在 **72 小时内** 向监管机构报告 (PIPL §57)
- 提供完整事件时间线 + 影响评估 + 整改方案

---

## 4. 跨境传输

### 4.1 数据本地化

- 默认: 全部数据存储在客户指定区域, 不出境
- 客户选择跨境部署 (如 AWS 美东) 时:
  - 必须签订标准合同条款 (SCCs)
  - 通过国家网信办安全评估
  - 通知所有数据主体并取得二次同意

### 4.2 政府访问

- 处理者收到任何政府/执法机构数据请求, **应立即通知控制者** (法律允许范围内)
- 仅在收到正式法律文书后才提供, 严格限于文书范围

---

## 5. 审计权利

控制者有权:
- 每年 1 次现场审计 (提前 30 天通知)
- 随时查询审计日志摘要 (`/api/audit/verify`)
- 委托独立第三方审计机构 (审计费由控制者承担, 整改费由处理者承担)

---

## 6. 终止与数据返还

合同终止 / 解除时:

| 阶段 | 操作 |
|---|---|
| T+0 ~ T+30 天 | 控制者通过 `/api/me/export` 全量导出, 验证完整性 |
| T+30 ~ T+90 天 | 处理者保留只读副本, 仅供控制者补充导出 |
| T+90 天 | 处理者执行 secure delete (NIST 800-88 standard) |
| T+120 天 | 处理者签署 **数据销毁证明** 并交付控制者 |

---

## 7. 责任与赔偿

- 处理者违约导致数据主体损失, 由处理者直接赔偿
- 罚款上限: 合同年费的 2 倍 (PIPL 实际罚款另计)
- 处理者保有 ≥ ¥500万 的数据安全责任险
