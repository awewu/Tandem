## 阅读说明

| 字段 | 含义 |
|---|---|
| ID | 功能编号 (类别号.子号) |
| 功能 | 功能名称 |
| 决策 | 是否做 / 怎么做 (创始人已拍板) |
| Phase | V1 (MVP) / V2 / V3 / ❌ (不做) |
| 工时 | XS<1w / S=1-2w / M=2-4w / L=1-2m / XL=2m+ |
| 依赖 | 前置依赖功能 ID |
| 备注 | 关键设计点 / 与宪章关联 |

**Phase 定义**:
- **V1 (MVP)**: 6-9 月内完成. 真正的"卖钱"产品. 必须包含核心差异化 + OKR 基础.
- **V2**: V1 后 6-9 月. 完整体验. 含集成生态.
- **V3**: V2 后 12+ 月. 高级 / 企业级 / 国际化.
- **❌**: 永不做 (违反宪章).

| 工时档 | 数量 | 单档估算 (人月) | 子小计 (人月) |

| XL | 4 | 3 | 12 |
| **合计** | **51** (V1, 不含已做的 12.1/12.3) | — | **~51 人月** |

**注**: 实际开发存在大量复用 (如 9.3 = 3.7, 5.1 = 6.1, 4.3 = 1.10), 实际净人月约 **35-45 人月**.

```
35-45 人月 ÷ 5-6 工程师 = 6-9 个月
```

**与 ROADMAP V1 目标完全对齐** (6-9 月).
