【2026-05-08 节点 · 路线澄清: 杀 Rocket.Chat, IM 自建在 Hermes 上】

  起因: 用户问 "为什么要 Docker, 不是挂架的 Hermes 驱动吗? 会不会影响 Hermes 联网进化?"

  复盘:
    - 架构事实: Hermes = 本地 Python CLI + Tauri Rust 命令, 是 agent runtime 底座.
                Tandem webui (本仓库) 通过 lib/hermes-api.ts 调 Hermes CLI 或本地 API 路由.
                没有 Docker, 也不需要 Docker — Hermes 直接跑在主机.
    - Docker 引入 Rocket.Chat 等于:
        (1) 多一套 Mongo 副本集运维成本
        (2) 上游 Rocket.Chat 分支同步 / SSO 桥接复杂度
        (3) Docker Desktop on Windows 走 WSL2, 1-2 GB 内存常驻
        (4) 哲学冲突: IM 应作为 Hermes 上的应用, 不是平行栈
    - "Hermes 联网进化" 实际是 docs/AGENT-FRAMEWORK.md §9 Hermes Watch 月报机制 (人 + 流程),
      不是后台进程. Docker 技术上对它 0 影响, 但路线上 Docker 拉远了 IM 与 Hermes 驱动的距离.

  决议: 杀 Rocket.Chat 路径, IM 继续自建 (Hermes runtime + Next.js + PG, 不引 Mongo/Docker).

  代码改动:
    ✓ docs/MANIFESTO.md §18              IM 从 "Rocket.Chat fork" 改为 "自建 (复用 Hermes + PG)", 列出 4 条理由
    ✓ docker-compose.tandem.yml          删 mongo / mongo-init-replica / rocketchat services + tandem_mongo_data volume
    ✓ docs/progress.txt                  本节点

  下一步 (IM 自建打磨):
