【2026-05-08 节点 · boot race 修 + 编码事故】

  问题: 重启验证 IM seed 时发现 `boot()` fire-and-forget seed, 首屏 race 读到空 store.

  修复:
    ✓ lib/boot.ts                       boot() 改 async, 暴露 _seedPromise; 同步路径 bootSync() 仅给 getRouter/getOrchestrator 用
    ✓ app/api/**/route.ts (27 callsites) boot();  → await boot();

  事故 + 复原:
    × 第一次批量改用 PowerShell Set-Content 默认编码 (CP936), 把 14 个文件的 UTF-8 中文注释烧成 mojibake (\uFFFD?), 偶尔吞换行 → 5 处 syntax error
    ✓ tsc 定位 syntax error 全修 (还原中文 + 补 \r\n)
    ✓ 50+ 处中文注释 / 错误信息字面量 best-effort 还原 (上下文猜测, 可能与原文有微差)
    ✓ 全文件 \uFFFD = 0, tsc 无新增错误 (剩余均 pre-existing: prisma-store IM repos / promotion-flow 类型 / e2e implicit any)

  教训 (写入纪律):
    - PowerShell 改文件必须 [System.IO.File]::WriteAllText($path, $content, $utf8NoBom), 严禁 Set-Content 不带 -Encoding UTF8
    - 项目无 git → 阶段 A 启动前必须先 `git init` + commit, 锁住业务代码
    - 批量替换前先在单文件试跑, 看输出再扩大范围
